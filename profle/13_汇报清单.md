## B. 你这个项目在做什么（问题定义与目标）

### B1. 总目标
- 任务：胸片（CheXpert）结构化报告生成/判别。
- 方法：冻结大语言模型（LLM）作为结构化语义空间，训练视觉端（ViT + Projector）把图像特征对齐到 JSON 输出。
- 输出：统一 UMS JSON（含 14 findings + study_view），并可程序化评估。

### B2. 你想要的结果
- 不只是“能生成 JSON”，而是要在 `present/absent/uncertain/null` 上有可用判别能力。
- 论文层面最终要和 ViT baseline 做对比，目标是接近或超过 baseline。

## F. 碰到的关键问题（完整复盘）

### F1. 数据先天稀疏（不是 bug）
- CheXpert 的 14 findings 在 30k 采样中 `blank/NaN≈70.47%`。
- 这意味着目标中大量字段本来就不可答，天然会稀释监督。

### F2. Prompt 设计与监督目标冲突
- 目前 prompt 强制输出“全量 14 条 findings + null 模板”。
- 这会给模型非常强的“输出 null 很安全”的先验。

### F3. 新发现：训练序列被截断（非常关键）
- 当前 `max_text_length=384`（见配置）。
- 实测（500 条样本）：
  - `prompt` 本身约 **259 tokens**；
  - `prompt+target` 平均约 **508 tokens**；
  - 训练时目标可用 token 只剩约 **125 tokens**（被截断）。
- 直接后果：后半标签（从 `Pneumonia` 到 `Support Devices`）几乎拿不到监督。
- 这也是为什么你会看到“mask 了也还是塌缩”。

### F4. 为什么 mask 了还是塌缩（直接回答）
- 不是“全掩码”，而是“部分 token 降权”：
  - `false_weight=0.1/0.2` 时，不可答状态 token 仍有梯度；
  - 即使 `false_weight=0`，如果 `keep_structural_tokens=true`，结构 token 仍在训练。
- 再叠加：prompt 强先验 + 截断 + 高比例 null 数据，模型仍倾向全 null。

### F5. 训练/部署层面的次要问题
- `flash_attn` 未安装：有性能损失，但不是塌缩主因。
- 远程下载（HF/ModelScope）曾不稳定：已通过镜像/重试绕过，不是当前主瓶颈。
