## C. 问题归因（1 页）
- 数据证据：CheXpert 30k 采样中 14 findings 的 `blank/NaN` 总体约 70.47%（UMS 中编码为 `state=null + answerability=false`），逐标签缺失见 `profle/schema/coverage_table.md`。
1. `null` 占比过高导致监督失衡。  
2. 固定 JSON 模板 token 过多，关键状态 token 梯度被稀释。  
3. 因此出现“格式正确但预测弱”的现象。

## D. V3 改动清单（代码级）
- `training/losses.py`：`StructuredLoss` 支持 token/sequence 加权。
- `training/trainer.py`：接入 `token_weighting`，训练和验证统一走加权 loss。
- `models/vivid_model.py`：`forward` 返回 `labels`，供加权损失计算。
- `scripts/train_cxr.py`：接入 `token_weighting` 参数，debug 自动禁用。
- `configs/cxr_chexpert_v3.yaml`：新增可直接训练的 v3 配置。

## E. 风险与对策（评委常问）

- 风险：仅靠 token weighting 不足以追平 baseline。  
  对策：下一步预留“字段级辅助损失”方案。

## H. 缺失率与解决路线（1页）
- 背景事实：CheXpert findings 标注稀疏（30k 里 `blank/NaN`≈70.47%），prompt 又要求输出全量 14 条，导致监督极不均衡。
- 口径：严格 UMS（blank=unknown，评估 mask）+ 对比口径（blank->absent，仅用于公平对比 baseline）。
- 路线 1（训练目标）：V3 token weighting（已落地）+ 下一步 answerability-aware loss（对 `answerable=false` 段落降权/屏蔽）。
- 路线 2（字段设计）：稀疏 findings 表示（present/absent/uncertain 列表）+ 后处理还原 14 label JSON 保持兼容。
- 路线 3（数据稠密）：teacher 伪标签（高置信度填充 blank）+ 小规模人工标注 calibration set（Codex 做标注工具）。
- 标注落地：`annotation_queue.csv` + `manual_labels.csv`（gitignore），优先 4 个高 missing 标签，各 50–100 张，20% 双人复标。

## I. V4 训练卡片（汇报后执行）
- V4 名称：`Answerability Masked Training`（在 V3 上增加 `answerability=false` 字段的 loss mask）。
- V4 目标：在不改变 UMS 语义的前提下，减少不可答字段对梯度的污染，提升有效分类监督。
- 与 V3 区别：V3 只调状态 token 权重；V4 增加“字段可答性”维度的结构化掩码。
- 结果读取：主看 `macro_f1/micro_f1/pred_nan_rate/per-label F1`，并保留双口径对比（严格 UMS + blank->absent）。
- 风险控制：若 `json_success_rate` 明显下降，先回滚 `false_weight`，不要先动 backbone/LLM 结构。
- 文档位置：V4 详细规范见 `profle/14_V4_AnswerabilityMaskedTraining.md`。
