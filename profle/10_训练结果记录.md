# 10 训练结果记录（更新至 2026-02-06）

## 1. 目标与当前状态
- 目标：用冻结 LLM 的结构化监督信号训练 ViT/Projector，输出可验证 JSON。
- 当前状态：核心架构未变，JSON 解析稳定性已修复，但分类性能仍显著低于 ViT baseline。

## 2. 关键实验结果（已完成）

说明：
- `json_success_rate=1.0` 说明结构化输出流程已通。
- 但 `pred_nan_rate` 高、`macro_f1` 低，说明模型在状态判别上仍偏“空/弱预测”。

## 3. 问题定位（当前结论）
1. **状态词监督不平衡**：训练目标里 `null` 比例高，导致模型倾向保守输出。  
2. **LM loss 稀释**：固定 JSON 模板 token 太多，关键状态 token（present/absent/uncertain/null）梯度占比不足。  
3. **现象一致**：解析成功不等于判别有效，当前瓶颈是“可解析”到“可判别”的迁移。

## 4. 已完成的 V3 代码改动（未启动训练）
本次只改代码，不启动训练。

### 4.1 状态词加权损失（核心）
- 文件：`training/losses.py`
- 改动：`StructuredLoss` 增加 state-aware token weighting 能力：
  - 支持 `token_id_weights`
  - 支持 `token_sequence_weights`
  - 支持 `default_token_weight`
- 作用：放大 `present/uncertain/absent` 的学习信号，抑制 `null` 过度主导。

### 4.2 训练器接入 V3 损失
- 文件：`training/trainer.py`
- 改动：
  - 新增 `token_weighting` 配置入口
  - 自动从 tokenizer 构建状态词权重（含多种 JSON 变体）
  - 训练/验证统一使用加权 token loss（若启用）
  - `autocast` 改为新接口优先，兼容旧接口
- 作用：无需改模型主结构即可切换到 V3 训练策略。

### 4.3 模型输出补充 labels（供加权损失使用）
- 文件：`models/vivid_model.py`
- 改动：`forward(return_dict=True)` 增加 `"labels"` 返回字段。

### 4.4 训练脚本接入
- 文件：`scripts/train_cxr.py`
- 改动：
  - 向 `VIVIDTrainer` 传入 `token_weighting`
  - debug 模式自动禁用 token weighting（保证调试轻量）

### 4.5 新增 V3 配置
- 文件：`configs/cxr_chexpert_v3.yaml`
- 关键参数：
  - `token_weighting.enabled: true`
  - `present: 3.0`
  - `uncertain: 2.5`
  - `absent: 1.5`
  - `null: 0.35`
- 训练输出目录：`outputs/cxr_chexpert_v3`

### 4.6 语法检查
- 已执行（未训练）：`py_compile` 检查通过。
