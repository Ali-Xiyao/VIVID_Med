# VIVID-Med v1.0 方案总览（GPT 版：详细拆分）


## 0. 一句话定位

**我们反过来用 LLM：不训练 LLM 去看医学图像，而是用冻结 LLM 作为“结构化语义解码器/监督空间”，训练 ViT 学到可迁移、可验证的医学视觉表征。**


## 0.1 论文故事线（主线方法的“新颖性抓手”）

**Semantic Knowledge Distillation（语义蒸馏）**：传统 one-hot 标签把疾病/属性当作彼此正交；冻结 LLM 的语义空间天然编码医学概念的相似性与共现拓扑。我们通过“结构化、受约束”的 token 对齐训练，让 **ViT 表征被蒸馏到 LLM 的语义流形上**，从而获得更强的跨域泛化与抗 shortcut 能力。

## 0.2 这句话是什么意思（给你自己/审稿人“秒懂版”）

把上面那句话拆开就是三件事：

1. **one-hot 的盲点**：标签把概念当作彼此独立的维度，只告诉模型“对/错”，不鼓励显式学到概念相似性与共现结构。
2. **LLM 的价值**：冻结 LLM 的表示空间里天然包含概念关系（距离/相似性/共现拓扑），可以视作一个“语义监督空间”。
3. **蒸馏的对象**：我们不是让 ViT 学会聊天，而是用结构化、受约束的 JSON token 目标去反向塑造 ViT 表征，使其经 projector 映射后能在该语义空间里“落到正确位置”。

> 这就是主故事线：**LLM 是训练时的老师（监督空间），ViT 是最终交付物（产物）**。

## 0.3 最终交付物（避免“瞎忙活”的关键一句话）

你最后交付的不是“更懂 LLM 的 ViT”，而是：

- **产物 A（主产物：表征）**：一个更强的 **medical ViT backbone checkpoint**（+ 可复现训练配方）。推理时输出 embedding/feature map；按需求接分类/分割/回归/检索等下游 head（LLM 不参与推理）。
- **产物 B（可选加分：端到端结构化）**：在 ViT 上再训练一个不含 LLM 的轻量 `UMS head/decoder`，让推理时直接输出 UMS 字段；输出仍经过 `schema + verifier` 做安全校验与 abstain（LLM 不参与推理）。


## 1. 你要让审稿人信的 3 个结论（全篇主线）

1. **LLM 是工具，ViT 是产品**：训练结束后丢掉 LLM，ViT-only 下游仍显著提升。
2. **结构化输出可验证，降低幻觉风险**：输出被约束在 UMS-JSON schema 内，并由 verifier 做单元测试式校验。
3. **模型真的看图（anti-shortcut 可量化）**：通过 image-swap / prompt-corruption / risk–coverage + 临床硬错误证明不是语言捷径。

### 1.1 三锤硬证据（把“价值”钉在实验里）

- **ViT-only transfer**：丢掉 LLM 后，external test / transfer / low-data 仍显著提升（证明“ViT 是产品”）
- **anti-shortcut**：image-swap / prompt-corruption（证明“必须看图”）
- **可靠性**：risk–coverage + clinical hard errors（laterality/unit/×10 等）（证明“医学价值/更安全”）


## 2. 三段式流水线（Figure 1 主图建议）

### (A) Offline：Verified Supervision 构建（数据轴）

公开数据（图像 + 标签/测量/元数据）→ **UMS-JSON**（含 answerability/uncertainty/provenance）→ **verifier** 输出 `pass/fail + failure_type + confidence + role`

核心原则（红线）：

- **凡是“可以从几何/元数据算出来的”，绝不让 LLM 编答案**；LLM 只做“问法/措辞”，不做真值。
- **`null` 是“不可答”的正式答案**：字段缺失/不可判定必须显式 `null`，并配套 answerability/uncertainty + role=abstain（细则见 `05_UMS_与Verifier_与FailureTaxonomy.md`）。

### (B) Online：LLM-mediated Structured Alignment 训练（训练轴）

`image → ViT(train) → projector(train) → frozen LLM(forward，可导到输入) → JSON tokens`

关键实现点：

- 冻结 LLM 参数 `requires_grad=False`，但 **forward 不能 `no_grad`**（需要对输入 embedding 可导，使梯度回传到 projector/ViT）。
- 训练用 teacher forcing（省算），推理/评测用 constrained JSON decoding（更严）。

### (C) Eval：ViT-only 迁移 + 可靠性评测（证据轴）

- **ViT-only transfer**：丢掉 LLM，仅用 ViT encoder 接 head/下游框架（分类/分割/回归）。
- **LLM swap**：固定 ViT+projector，替换 LLM，检验稳定性。
- **可靠性指标**：ECE/Brier、risk–coverage、clinical hard errors（laterality/measurement 等）。

## 3. 最小可投稿配置（v1.0 默认）

数据（v1.0 推荐“可录用组合”，含 external test / transfer）：

- **CXR 训练：CheXpert**；**CXR 外部测试：NIH ChestX-ray14**（domain shift）
- **CT 训练：AMOS（CT 部分）**；**CT 外部迁移：KiTS21**（任务与来源不同）
- （可选加分）**测量：LIDC-IDRI**（数值字段 + unit verifier 很契合）

模型与算力（对话默认可行配置）：

- GPU：RTX 3090 24GB（或同级）
- backbone：ViT-B/16（或 ViT-L）
- precision：fp16/bf16
- 训练对象：ViT + projector（LLM 冻结）

模型（v1.0 默认口径，细节见 `04_模型.md`）：

- **ViT**：CXR 用 `ViT-B/16`；CT 推荐 **2.5D 切片法**（v1.0 默认：相邻 **3** 切片多通道输入）
- **Projector**：`2-layer MLP + learnable prefix tokens`（v1.0 默认：224 + prefix=16；384/32 作为 v1.1/ablation）
- **冻结 LLM（teacher space）**：
  - 训练：`Meta-Llama-3.1-8B-Instruct`
  - swap：`Qwen2.5-7B-Instruct`
- **现成 VLM**：不作主线，但应作为 baseline（`LLaVA-Med`、`RadFM` 等）

## 3.1 医学场景怎么讲“为什么要改 ViT”（性价比与通用性）

医疗里常见的真实矛盾是：

- **大模型很强，但临床推理部署很难**（算力/延迟/合规/成本）
- **跨域泛化是常态问题**（换医院/机型/协议就掉点）
- **致命错误比平均指标更重要**（laterality/unit/×10 等）

因此我们讲的不是“改 ViT 很酷”，而是：

> 用懂语义的老师（冻结 LLM）在训练期提供监督，把语义结构压进可部署的学生（ViT）；部署期只用 ViT，但更稳、更可控、更少硬错误。

补充一条很重要的训练直觉（也能做成消融加分项）：
- **teacher 不一定越大越好**：teacher 过强可能“替学生做题”，导致 ViT 学不到；建议做 teacher capacity ablation 找 sweet spot（见 `07_实验矩阵与评估协议.md`）。

## 3.2 必做对照与“翻车预案”（写在论文里就不怕 reviewer 追问）

- **必须对照：task-head baseline**（同数据同 backbone 直接接分类/分割/回归头训练）  
  不比它，最容易被质疑 busywork（见 `07_实验矩阵与评估协议.md`）。
- **Plan A（理想）**：你在 external/low-data/可靠性上显著赢 task-head baseline，同时同分布不落后太多。
- **Plan B（可接受且很医学）**：task-head baseline 在同分布满数据更强，但你在 external/low-data/硬错误/拒答可靠性更强 → 主张写成“面向临床部署的鲁棒与安全表征”。

如果你担心“比不过 BiomedCLIP/大规模图文预训”：
- 不硬拼“通用 VLM 能力”，把主表战场放在 **ViT-only + external/low-data/可靠性**；
- 额外加一行很稳的实验：**BiomedCLIP init + Ours**，证明你贡献的是训练机制，而不是数据规模（见 `07_实验矩阵与评估协议.md`）。

## 3.3 训练时输入/输出一句话（避免“到底在训什么”）

训练输入是 `(image, prompt)`；监督目标是 **UMS-JSON 的结构化 token 序列**（含 answerability/abstain），模型输出是 next-token logits（+ 可选 answerability head），梯度只更新 ViT+projector（细节见 `02_方法.md`、`05_UMS_与Verifier_与FailureTaxonomy.md`、`06_训练目标与SamplePolicy.md`）。
