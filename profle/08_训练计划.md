## 2. 模型选择

### 2.1 Vision Encoder (ViT)

| 配置项 | 选择 | 说明 |
|-------|------|------|
| 模型 | `vit_base_patch16_224` | ViT-Base，patch size 16，输入 224×224 |
| 预训练 | ImageNet-21k | 使用 timm 库加载预训练权重 |
| 输出类型 | `cls` | 使用 [CLS] token 作为图像表征 |
| 参数量 | ~86M | **可训练** |

**选择理由**：
- ViT-Base 是医学影像领域常用的 backbone，平衡了性能和计算成本
- ImageNet 预训练提供良好的初始化
- 使用 [CLS] token 简化后续 projector 设计

### 2.2 Projector

| 配置项 | 选择 | 说明 |
|-------|------|------|
| 结构 | 2-layer MLP | 768 → 1536 → 1536 |
| 前缀 tokens | 16 | 可学习的 prefix tokens |
| Dropout | 0.1 | 防止过拟合 |
| 参数量 | ~7M | **可训练** |

**作用**：将 ViT 的 768 维特征投影到 LLM 的 1536 维空间

### 2.3 LLM (冻结)

| 配置项 | 选择 | 说明 |
|-------|------|------|
| 模型 | `Qwen/Qwen2.5-1.5B-Instruct` | Qwen2.5 1.5B 指令微调版 |
| 精度 | BFloat16 | 节省显存 |
| 参数量 | ~1.5B | **冻结，不训练** |
| 下载源 | ModelScope（魔搭社区） | 国内镜像，下载更快 |

**选择理由**：
- 1.5B 参数量适合 24GB 显存
- Qwen2.5 对中文和结构化输出支持较好
- 指令微调版本更适合生成任务

### 2.4 参数统计

| 组件 | 参数量 | 状态 |
|------|--------|------|
| ViT | ~86M | 可训练 |
| Projector | ~7M | 可训练 |
| LLM | ~1.5B | 冻结 |
| **总可训练** | **~93M** | |
| **总参数** | **~1.6B** | |

## 3. 训练方式

### 3.1 核心架构

```
Image → ViT(train) → Projector(train) → Frozen LLM(forward) → JSON token logits
         ↑                ↑                    ↓
      梯度回传          梯度回传            计算 loss
```

**关键点**：
- LLM 参数冻结 (`requires_grad=False`)
- 但 LLM forward 必须保持可导（不能用 `torch.no_grad()`）
- 梯度通过 LLM 回传到 Projector 和 ViT

### 3.2 训练目标

v1.0 使用单一损失函数 `L_tok`（Next-Token Prediction）：

```
L = L_tok = CrossEntropy(logits, target_tokens)
```

其中：
- `logits`: LLM 输出的 token 概率分布
- `target_tokens`: UMS-JSON 的 token 序列

**后续可扩展**（v1.1+）：
- `L_rank`: Hard-negative ranking loss
- `L_vdep`: Visual-dependence anti-shortcut loss
- `L_ans`: Answerability prediction loss

### 3.3 优化器配置

| 配置项 | 值 | 说明 |
|-------|-----|------|
| 优化器 | AdamW | 带权重衰减的 Adam |
| 学习率 | 1e-4 | |
| 权重衰减 | 0.01 | |
| Warmup | 500 steps | 线性 warmup |
| 调度器 | Cosine | 余弦退火 |

### 3.4 批次配置

| 配置项 | 值 | 说明 |
|-------|-----|------|
| Batch size | 8 | 单卡 batch |
| 梯度累积 | 4 | 有效 batch = 32 |
| 最大步数 | 10,000 | |
| 混合精度 | BFloat16 | 节省显存 |

### 3.5 显存估算（24GB GPU）

| 组件 | 显存占用 |
|------|---------|
| LLM (BF16) | ~3 GB |
| ViT + Projector | ~0.5 GB |
| 激活值 + 梯度 | ~8-12 GB |
| **预计总占用** | **~12-16 GB** |

## 8. 注意事项

1. **首次运行会下载 LLM**：Qwen2.5-1.5B 约 3GB，从 ModelScope 下载
2. **显存不足时**：减小 batch_size 或增加 gradient_accumulation_steps
3. **训练中断**：使用 `--resume` 从检查点恢复
4. **监控训练**：观察 loss 曲线，正常应该稳定下降
