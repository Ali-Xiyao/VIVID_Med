## 0. v1.0 拍板条款（默认配置，优先保证闭环可跑）

- **[拍板 v1.0]** 缺失字段统一输出 `null`，并在字段级 `answerability.<field>=false`（`study_view` 不引入 `unknown/U` 等新枚举）。
- **[拍板 v1.0]** confusable pairs 仅用于 hard negatives / ranking（`L_rank`），**不**作为 verifier 的互斥/一致性约束（verifier 只保留硬矛盾与单值字段互斥）。
- **[拍板 v1.0]** LLM swap 协议：freeze ViT 与两侧 LLM，仅用 1k–5k 样本训练 `projector_swap` 适配新 LLM（证明接口可插拔；不宣称零成本 swap；也不强求 tokenizer 一致）。
- **[拍板 v1.0]** v1.0 默认分辨率 224 跑通闭环；384 与 token-drop/VRL 作为 v1.1/ablation（不作为 v1.0 默认）。

## B. 模型与实现细节

- Frozen LLM 选型（已确定）：
  - 训练 LLM：`meta-llama/Llama-3.1-8B-Instruct`
  - swap LLM：`Qwen/Qwen2.5-7B-Instruct`
- tokenization 与 prompt 格式（已拍板 v1.0）：
  - 不要求统一 tokenizer；prompt 文本层统一字段名/JSON schema 即可
  - constrained decoding / grammar / trie 按 **每个 LLM 的 tokenizer 单独生成**（swap 时同理）
- projector 形式（已拍板 v1.0）：
  - `2-layer MLP + learnable prefix tokens`
  - prefix tokens：v1.0 默认 **16（224）**；v1.1/ablation 可用 **32（384）**
- constrained decoding 实现（已确定大方向）：grammar/FSM（或 trie）约束；细节实现待定
- 训练工程开关（已拍板 v1.0 默认）：
  - `bf16`（或 fp16）+ flash-attn + gradient checkpointing：默认开启（以显存优先）
  - 不建议训练期 4-bit（容易踩“可导到输入 embedding”的实现坑）；若显存不够优先尝试 8-bit weights + bf16 compute（以框架支持为准）
  - 推理/验证阶段可使用 4-bit（不影响训练梯度路径）

## C. 训练超参（建议你最终固定在一个表格里）

- 输入分辨率（已拍板 v1.0）：**224**（384 作为 v1.1/ablation）
- batch size（建议 v1.0 默认）：per-GPU **4**（再用 grad accumulation 做到 global batch ~64）
- 优化器（建议 v1.0 默认）：AdamW（`weight_decay=0.05`）
- 学习率（建议 v1.0 默认，分组 LR）：
  - `lr_vit = 1e-5`
  - `lr_proj = 1e-4`
  - schedule：cosine + 3% warmup
- 训练步数（建议 v1.0 默认）：**8k steps**（足够闭环与写作；更长可做附录/补充）
- hard negatives 数量（已拍板 v1.0）：每样本 **4 个**
- loss 权重（v1.0 分阶段；v1.1 再补充消融）：
  - v1.0 先只跑 `L_tok` 跑通（要求 100% 可解析 + verifier pass rate 上来）
  - v1.1 第二阶段：`λ_rank=0.2`，margin `m=0.2`
  - v1.1 第三阶段：`λ_vdep=0.1`（最后再加）
