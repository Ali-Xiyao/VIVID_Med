# 模型

> 本文件定义 v1.0 默认模型配置与 baseline 口径；数据集选择与字段覆盖见 `03_数据集.md`。

## 1. 默认模型（v1.0）

主线模型（最终版；主线必须自己搭：ViT + Projector + Frozen LLM）：

- **视觉编码器（ViT）**：CXR 用 `ViT-B/16`；CT 推荐 **2.5D 切片法**（v1.0 默认：相邻 **3** 切片当 3 通道输入，同样用 `ViT-B/16`）
- **Projector**：`2-layer MLP + learnable prefix tokens`，把 ViT 特征映射到 LLM hidden dim，作为视觉前缀 token 拼到文本 prompt 前v1.0 默认：输入分辨率 224 + prefix tokens=16（384/32 作为 v1.1/ablation）
- **冻结 LLM（训练用 + swap 用）**： 4B
  - 训练 LLM：`Meta-Llama-3.1-8B-Instruct`
  - swap LLM：`Qwen2.5-7B-Instruct`

关键技术点（正文必须写清楚）：

- 冻结 LLM 参数，但 forward **必须保持对输入 embedding 可导**（不能用 `torch.no_grad()` 包住 LLM forward）

LLM swap（v1.0 口径，工程更稳）：

- swap 时不强求 tokenizer 一致；每个 LLM 各自使用自己的 tokenizer 与约束 grammar。
- swap 协议定义为：**freeze ViT（表征固定），仅用少量数据（1k–5k）训练 `projector_swap`** 去适配新的 LLM embedding space（不宣称零成本 swap）。

## 2. 现成 VLM（baseline）

- **不用作主线**（会削弱“LLM 是监督空间，ViT 是产物”的叙事）
- **必须作为 baseline**：`LLaVA-Med`、`RadFM`（尽量抽视觉 encoder 做同样 ViT-only 下游；不可抽取则说明等价对比口径）
