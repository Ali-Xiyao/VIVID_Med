# VIVID-Med V9：数据与实验

## 1. 数据集总览

| 数据集 | 模态 | 用途 | 样本量 | 标签 |
|--------|------|------|--------|------|
| CheXpert 30k | CXR | 预训练 + 分类 transfer | 30,000 | 14 findings（全量）/ 7 findings（Top7 子集） |
| AMOS 2.5D | CT | 预训练 | 26,457 train / 13,788 val | 15 器官 |
| KiTS21 2.5D | CT | 分割 transfer | 45,924 train / 11,483 val | kidney/tumor/cyst |
| NIH ChestX-ray14 | CXR | External test | 25,596 | 7 共同标签 |

> **注意**：当前 V9 SVRL 预训练仅使用 7 个 CXR label（Top7），但 14-label transfer 评估已证明优势更大。建议用全部 14 label 重新预训练（见第 5 节问题 1）。

### 1.1 CheXpert

- 来源：CheXpert-v1.0-small（Stanford）
- Top7 标签：Atelectasis, Cardiomegaly, Consolidation, Edema, Lung Opacity, Pleural Effusion, Support Devices
- 标签处理：blank/NaN → state=null + answerability=false；uncertain → state=uncertain
- UMS 文件：`data/dataset/processed/chexpert_ums_train.jsonl`

### 1.2 AMOS 2.5D

- 来源：AMOS2022（CT 部分，241 train + 120 val volumes）
- 2.5D 处理：相邻 3 axial slices → 3 通道输入
- CT windowing：soft tissue (W=400, L=40)，归一化 [0,1]
- 只保留含非背景器官的 slices
- 15 器官类别（spleen, kidney_R/L, gallbladder, esophagus, liver, stomach, aorta, IVC, pancreas, adrenal_R/L, duodenum, bladder, prostate/uterus）
- UMS 文件：`data/dataset/processed/amos_ums_{train,val}.jsonl`

### 1.3 KiTS21 2.5D

- 来源：KiTS21（300 cases，前 240 train / 后 60 val）
- 同样 2.5D + CT windowing 处理
- 4 类分割：background(0), kidney(1), tumor(2), cyst(3)
- 用于 transfer evaluation，不参与预训练
- UMS 文件：`data/dataset/processed/kits_ums_{train,val}.jsonl`

### 1.4 NIH ChestX-ray14

- 来源：NIH Clinical Center
- 仅用于 external test（不参与训练）
- 与 CheXpert 共享 7 个标签用于跨域评估

### 2.1 分类 Transfer 详细结果

#### 7-label + dense_top_k=10000 口径

| 实验 | macro_F1 | macro_AUC | micro_F1 |
|------|---:|---:|---:|
| Baseline (ImageNet) | 0.8839 | 0.8473 | 0.8974 |
| From-VIVID V8 | 0.8790 | 0.8475 | 0.8963 |
| From-VIVID V9 SVRL | 0.8757 | 0.8530 | 0.8977 |

- V9 在 7-label 口径下 macro_F1 略低于 baseline，但 macro_AUC 有提升（+0.0057）
- 7-label 口径只覆盖高频标签，不能充分体现 SVRL 的优势

#### 14-label 全量口径（无 dense_top_k 筛选，~29k 训练样本）⭐ 主表口径

| 实验 | macro_F1 | macro_AUC | micro_F1 | val_loss |
|------|---:|---:|---:|---:|
| Baseline (ImageNet) | 0.8987 | 0.7927 | 0.8925 | 0.2521 |
| From-VIVID V9 SVRL | **0.9064** | **0.8210** | **0.8978** | **0.2455** |
| 差值 | **+0.0077** | **+0.0283** | **+0.0053** | -0.0066 |

- **14-label 全量口径下 V9 SVRL 全面优于 baseline**
- macro_AUC 提升 +2.83 个百分点，排序能力显著更强
- 低频/难分类标签改善最大：
  - Pneumothorax: F1 0.352→0.426 (+0.074), AUC 0.737→0.763 (+0.026)
  - Pleural Other: AUC 0.279→0.475 (+0.197)
  - Pneumonia: AUC 0.655→0.699 (+0.044)
  - Support Devices: AUC 0.599→0.652 (+0.053)
  - Fracture: AUC 0.824→0.879 (+0.055)

#### 关键发现

1. **14-label 全量 > 7-label dense10k 作为主表口径**：全量口径更能体现 SVRL 在长尾分布上的优势
2. **AUC 提升 > F1 提升**：说明 SVRL 学到的表征在概率排序上更好，不仅仅是阈值分类
3. **低频标签受益最大**：SVRL 预训练通过结构化 JSON 预测，迫使 ViT 编码更全面的视觉信息

## 3. 评估协议

### 3.1 分类 Transfer（主指标）

1. 提取 ViT backbone（丢弃 Projector + LLM）
2. 接线性分类头，fine-tune on CheXpert dense10k
3. 评估：CheXpert val + NIH external test
4. 指标：macro_F1, micro_F1, macro_AUC, per-label F1/AUC
5. 3 seeds (0,1,2)，报告 mean ± std

### 3.2 分割 Transfer（次要指标）

1. 提取 ViT backbone
2. 接 ViTSegHead（4×ConvTranspose2d），fine-tune on KiTS21
3. 指标：Dice (kidney, tumor), HD95
4. Low-data 曲线：10%, 25%, 50%, 100%

### 3.3 可靠性评测

- **校准**：ECE, Brier score
- **风险-覆盖**：coverage vs error 曲线，hard error rate @ 90% coverage
- **临床硬错误**：laterality_error, unit_error, measurement_hard_error
- **anti-shortcut**：prompt corruption sensitivity, image swap sensitivity

### 3.4 公平性对齐

- 相同 backbone、相近训练步数、相同数据规模
- 报告 GPU 型号、训练时长、显存、精度
- 统一评估口径

## 4. 翻车预案

- **Plan A**：external/low-data/可靠性显著赢 task-head baseline → 主线最强
- **Plan B**：task-head baseline 同分布更强，但我们 external/low-data/硬错误更强 → "面向临床部署的鲁棒表征"

## 5. 当前存在的问题与待决策项

### 问题 1：V9 预训练只用了 7 个 CXR label

- 当前 V9 SVRL 预训练时 CXR 数据只使用 7 个 selected_labels
- ViT 在预训练阶段对 Pleural Other、Support Devices、Fracture 等 7 个 label 的视觉模式完全没见过
- 但 14-label transfer 已经全面超过 baseline，说明 SVRL 表征有泛化性
- **建议**：用全部 14 label 重新预训练，预期能进一步提升
- **代价**：需要重跑 10000 steps（~4-6 小时 on RTX 3090）

### 问题 2：结果提升幅度是否足够

- 14-label macro_F1 +0.77%，macro_AUC +2.83%
- 对 MICCAI 来说 AUC +2.83% 是 decent 但不算 overwhelming
- 需要通过消融 + low-data + 跨域来放大优势
- 可能的提升方向：Multi-Scale Token Aggregation (MSTA)（见 01_方案与方法.md）

### 问题 3：已排除的改进方向

以下方向经讨论后排除：

1. **Uncertainty-Aware Structured Prediction**：CheXpert 的 uncertain 标签多为缺失（blank/NaN），不是真正的不确定性。V3/V4 已尝试 token weighting（present:3.0, uncertain:2.5, absent:1.5），效果不好。
2. **Attention-Guided Spatial Grounding Loss**：CXR 是 2D 投影，85% 样本有多个 finding 共存，空间高度重叠（Lung Opacity+Edema 共现 22%，Lung Opacity+Support Devices 共现 30%）。硬做互斥 diversity loss 会伤害性能。
3. **Adaptive Mask Ratio**：工程复杂度高（需要滑动平均/趋势检测，设上下限），novelty 偏弱。

### 问题 4：评估协议公平性确认

- 三组实验（Baseline / V8 / V9）在 7-label 口径下使用完全相同的数据和超参
- 14-label 口径下 Baseline 和 V9 使用相同配置，仅 ViT 初始化不同
- Transfer 协议：backbone_lr=1e-5, head_lr=1e-3（分组学习率）

### 问题 5：KiTS21 分割 Dice 极低（⚠️ 需要解决）

#### 现象

Baseline 和 V9 SVRL 的 KiTS21 分割结果都极低（mean_dice_fg ≈ 0.287），cyst 类完全为 0。两者几乎无差异。

#### 数据集类别分布分析

| 类别 | Train 切片数 | 占比 | Val 切片数 | 占比 |
|------|---:|---:|---:|---:|
| kidney (class 1) | 45,924 | 100% | 11,483 | 100% |
| tumor (class 2) | 19,144 | 41.7% | 5,193 | 45.2% |
| cyst (class 3) | 5,625 | 12.2% | 1,363 | 11.9% |

#### 原因分析

1. **Cyst 类严重不平衡**：仅 12.2% 切片含 cyst，且 cyst 区域通常很小。模型学会忽略 cyst，全部预测为 background/kidney/tumor
2. **Loss 无类别加权**：当前使用 `CrossEntropyLoss(ignore_index=255)`，没有 class weights 来补偿不平衡
3. **训练步数可能不足**：3000 steps（effective batch=16），对 45k 训练样本仅约 1 epoch
4. **SegHead 架构简单**：4 层 ConvTranspose2d 直接上采样，没有 skip connection 或多尺度特征
5. **与 ViTP 的差距**：ViTP 在 AMOS2022 上达到 mDice=90.60%，使用的是 ViT-Adapter + UperNet（远比我们的 SegHead 复杂）

#### 改进方向

1. 添加 class weights 到 CrossEntropyLoss（weight_cyst >> 1.0）
2. 使用 Dice Loss + CE Loss 组合
3. 增加训练步数到 10000+
4. 考虑更强的分割头（UperNet 或 ViT-Adapter）
5. 对 cyst 类做过采样或 hard example mining

#### 对论文的影响

当前分割结果不适合放入论文。需要先解决上述问题，或者将分割实验定位为"初步探索"而非核心贡献。CMSA novelty 的验证需要分割结果有意义才行。

### 问题 6：ViTP v3 论文更新对比

ViTP (Li et al. 2025, arXiv 2509.17562v3) 的关键结果：

| 数据集 | ViTP | nnU-Net | MedMAE |
|--------|---:|---:|---:|
| AMOS2022 (CT) | **90.60** | 87.28 | 90.11 |
| BraTS2021 (MRI) | **72.11** | 71.03 | 69.59 |
| CovidQUEx (X-ray) | **91.55** | 90.41 | 90.18 |

ViTP 与我们的关键差异：
- ViTP 训练 LLM（end-to-end），我们冻结 LLM
- ViTP 用 0.5B LLM（更小更好），我们用 1.5B
- ViTP 用 random 75% token drop，我们用 GFTM（attention-guided）
- ViTP 分割用 ViT-Adapter + UperNet，我们用简单 ConvTranspose2d SegHead
- ViTP 在 8×A40 上训练 ~23h，我们在 2×3090 上训练 ~6h
- ViTP 没有 consistency loss 和跨模态统一训练
