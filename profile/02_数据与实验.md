## 1. 数据集总览

| 数据集           | 模态 | 用途                   | 样本量                    | 标签                                         |
| ---------------- | ---- | ---------------------- | ------------------------- | -------------------------------------------- |
| CheXpert 30k     | CXR  | 预训练 + 分类 transfer | 30,000                    | 14 findings（全量）/ 7 findings（Top7 子集） |

### 1.1 CheXpert

- 来源：CheXpert-v1.0-small（Stanford）
- Top7 标签：Atelectasis, Cardiomegaly, Consolidation, Edema, Lung Opacity, Pleural Effusion, Support Devices
- 标签处理：blank/NaN → state=null + answerability=false；uncertain → state=uncertain
- UMS 文件：`data/dataset/processed/chexpert_ums_train.jsonl`

### 1.2 AMOS 2.5D

- 来源：AMOS2022（CT 部分，241 train + 120 val volumes）
- 2.5D 处理：相邻 3 axial slices → 3 通道输入
- CT windowing：soft tissue (W=400, L=40)，归一化 [0,1]
- 只保留含非背景器官的 slices
- 15 器官类别（spleen, kidney_R/L, gallbladder, esophagus, liver, stomach, aorta, IVC, pancreas, adrenal_R/L, duodenum, bladder, prostate/uterus）
- UMS 文件：`data/dataset/processed/amos_ums_{train,val}.jsonl`

### 1.3 KiTS21 2.5D

- 来源：KiTS21（300 cases，前 240 train / 后 60 val）
- 同样 2.5D + CT windowing 处理
- 4 类分割：background(0), kidney(1), tumor(2), cyst(3)
- 用于 transfer evaluation，不参与预训练
- UMS 文件：`data/dataset/processed/kits_ums_{train,val}.jsonl`

### 1.4 NIH ChestX-ray14

- 来源：NIH Clinical Center
- 仅用于 external test（不参与训练）
- 与 CheXpert 共享 7 个标签用于跨域评估

### 2.4 关键发现

1. **核心贡献是 Frozen LLM Supervision 框架本身**：No mask+No consistency（纯框架）AUC=0.8563（vs baseline 0.7927，+6.4 pp），无需额外组件
2. **GFTM、Consistency、CT 数据均未带来正面贡献**：所有消融都优于 SVRL-full，组件叠加产生干扰
3. **混合 CT 数据反而拖累 CXR 分类 transfer**：-CT（仅 CXR）F1=0.9149 / AUC=0.8446，均优于 SVRL-full（CXR+CT 混合）F1=0.9064 / AUC=0.8210（F1 +0.85 pp，AUC +2.36 pp），CMSA 跨模态对齐 novelty 不成立
4. **GFTM 是预训练过拟合的主因**：含 GFTM 的实验 best@500-1000 后退化，不含 GFTM 的训练健康（best@5000-6500）
5. **AUC 提升 > F1 提升**：LLM supervision 学到的表征在概率排序上更好

### 2.5 论文叙事方向

- 原方案"三个 novelty（GFTM + FSA + CMSA）"已不成立
- 核心贡献重新定位为：**Frozen LLM Supervision for Visual Representation Learning**
- -CT 结果（F1=0.9149, AUC=0.8446）反超 SVRL-full（F1=0.9064, AUC=0.8210），F1 +0.85 pp，AUC +2.36 pp → 混合 CT 数据不仅无益反而有害，CMSA novelty 不成立
- 所有消融均优于或持平 SVRL-full → GFTM、Consistency、CMSA 三个组件均无正面贡献
- 消融结果本身有价值：证明简单框架优于复杂组合，为后续研究提供指导

### 2.6 V10 新模块设计

基于 V9 消融结论（纯框架最优，GFTM hard masking 有害），V10 提出两个轻量级增强模块：

**SAR (Soft Attention Reweighting)**：替代 GFTM 的 hard masking
- 用 CLS-to-patch attention 的倒数作为软权重，放大低注意力 tokens
- 公式：`token_i = token_i * (1 + α * (w_i - 1))`，其中 `w_i = normalize(1/attn_i)`
- α 为可学习参数（初始值 1.0），训练时自动调节加权强度
- 不丢弃任何 token → 避免 GFTM 的信息损失和过拟合
- 实现：`models/vit.py` 中 `_apply_sar()` 方法

**HFP (Hierarchical Feature Projection)**：多层 ViT 特征融合
- 提取 ViT 第 4/8/12 层（浅层纹理 + 中层结构 + 深层语义）
- 每层一个轻量 MLP projector → concat → fusion layer
- 让 frozen LLM supervision 的梯度传到 ViT 所有层，而非仅最后几层
- 实现：`models/projector.py` 中 `HierarchicalProjector` 类

配置文件：
- `configs/v10_sar_hfp.yaml` — SAR + HFP 联合
- `configs/v10_sar_only.yaml` — SAR 消融
- `configs/v10_hfp_only.yaml` — HFP 消融

### 2.7 V10.1 SPD 模块设计

**SPD (Structured Prediction Decomposition)**：多组 cross-attention query tokens 分解视觉表征

- 3 组可学习 query tokens（每组 2 个），通过 cross-attention 从 ViT 特征中提取不同方面信息
- 与 VisionProjector 的区别：prefix tokens 不依赖图像内容 → SPD query tokens 自适应提取
- 与 HFP 的区别：HFP 纵向多尺度（多层融合）→ SPD 横向多方面（单层分解）
- 正交性损失（ortho_weight=0.01）：鼓励各组 attention pattern 互不重叠
- 实现：`models/spd.py` 中 `SPDProjector` 类
- 配置：`configs/v10.1_spd.yaml`（独立于 SAR/HFP，不叠加）

### 2.8 评估协议升级计划

> 决策日期：2026-02-15

**问题**：当前 full fine-tune 评估下 baseline F1=0.8987 已经很高，预训练方法的提升空间被压缩（SVRL vs baseline 仅 +0.77pp F1）。Full fine-tune 让 ViT 所有参数在下游任务上充分调整，稀释了预训练表征质量的差异。

**解决方案**：采用表征学习领域标准评估矩阵（SimCLR/DINO/MAE/ViTP 均采用）：

| 评估方式 | 说明 | 预期 baseline | 预期差距 | 优先级 |
| --- | --- | --- | --- | --- |
| Linear probe | 冻结 ViT，只训练线性分类头 | F1≈0.70-0.75 | 5-10pp | ⭐ 主表 |
| Low-data fine-tune | 10%/25%/50% 训练数据 | F1≈0.82-0.85 | 3-5pp | ⭐ 数据效率曲线 |
| Cross-domain | NIH ChestX-ray14（7 共同标签） | — | — | ⭐ 泛化能力 |
| Full fine-tune | 当前方式（backbone_lr=1e-5） | F1≈0.90 | ~1pp | 补充 |

Linear probe 下 ImageNet baseline 会大幅下降（自然图像特征对医学任务直接可用性差），而 SVRL 预训练的医学表征应有显著优势。

**Linear probe 超参数（计划）**：

| 参数 | 值 | 说明 |
| --- | --- | --- |
| backbone_lr | 0 | 冻结 ViT |
| head_lr | 1e-3 | 线性分类头 |
| max_steps | 3000 | 冻结 backbone 收敛更快 |
| 其他 | 同 full fine-tune | — |

### 2.9 CMSA（跨模态混合训练）决策

> 决策日期：2026-02-15

**结论：不作为 contribution 提出。**

消融数据明确显示混合 CT 数据有害：
- -CT（仅 CXR）：F1=0.9149, AUC=0.8446
- SVRL-full（CXR+CT）：F1=0.9064, AUC=0.8210
- 差距：F1 -0.85pp, AUC -2.36pp

当前实现仅有混采样（`data/multi_modal_loader.py`），无显式跨模态对齐损失，技术贡献不足。

**论文处理方式**：在消融实验部分作为 negative finding 报告，不作为 contribution。"We investigated cross-modal pre-training by mixing CXR and CT data. Results show naive mixing degrades CXR transfer, suggesting modality-specific representations are more effective under our framework."

## 4. 已排除的改进方向

1. **Uncertainty-Aware Structured Prediction**：CheXpert uncertain 标签多为缺失，V3/V4 已尝试 token weighting 效果不好
2. **Attention-Guided Spatial Grounding Loss**：CXR 2D 投影，85% 样本多 finding 共存，空间高度重叠
3. **Adaptive Mask Ratio**：工程复杂度高，novelty 偏弱

## 5. ViTP 对比

ViTP (Li et al. 2025, arXiv 2509.17562v3)：

| 数据集 | ViTP | nnU-Net | MedMAE |
| --- | ---: | ---: | ---: |
| AMOS2022 (CT) | **90.60** | 87.28 | 90.11 |
| BraTS2021 (MRI) | **72.11** | 71.03 | 69.59 |
| CovidQUEx (X-ray) | **91.55** | 90.41 | 90.18 |

关键差异：ViTP 训练 LLM（end-to-end）+ 0.5B LLM + random 75% token drop + ViT-Adapter+UperNet + 8×A40。我们冻结 LLM + 1.5B + SAR 软加权（V10）/ SPD 分解（V10.1）+ 简单 SegHead + 2×3090。
