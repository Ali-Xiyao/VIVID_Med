# VIVID-Med：方案与方法

> 最后更新：2026-02-15

## 1. 定位

**用冻结 LLM 作为"结构化语义监督空间"，训练 ViT 学到可迁移、可验证的医学视觉表征。训练后丢弃 LLM，只交付 ViT backbone。**

核心叙事（Semantic Knowledge Distillation）：传统 one-hot 标签把疾病当作彼此正交；冻结 LLM 的语义空间天然编码概念相似性与共现拓扑。我们通过结构化 token 对齐训练，让 ViT 表征被蒸馏到 LLM 的语义流形上。

> **LLM 是训练时的老师，ViT 是最终交付物。**

## 2. 核心发现（基于 V9 消融实验）

V9 SVRL 设计了三个组件（GFTM、FSA Consistency、CMSA），但消融实验表明：

| 结论 | 证据 |
| --- | --- |
| **核心贡献是 Frozen LLM Supervision 框架本身** | No mask+No consistency（纯框架）AUC=0.8563（vs baseline 0.7927，+6.4pp） |
| GFTM hard masking 有害 | 含 GFTM 实验 best@500-1000 后退化；不含 GFTM 训练稳定（best@5000-6500） |
| Consistency loss 无正面贡献 | -Consistency 消融 F1=0.9130, AUC=0.8439，均优于 SVRL-full |
| 混合 CT 数据反而拖累 CXR transfer | -CT（仅 CXR）F1=0.9149, AUC=0.8446，优于 SVRL-full（CXR+CT）F1=0.9064, AUC=0.8210 |
| 所有消融均优于 SVRL-full | 组件叠加产生干扰，简单框架优于复杂组合 |

**论文贡献重新定位**：不再以 GFTM/FSA/CMSA 为 novelty，核心贡献是 Frozen LLM Supervision 框架 + 新增模块（V10 SAR+HFP 或 V10.1 SPD）。

## 3. V9 SVRL 方法

### 3.1 架构

```text
Image → ViT(trainable) → Projector(trainable) → Frozen LLM(forward) → JSON token logits
```

- ViT：vit_base_patch16_224，输出 all tokens（197 = 1 CLS + 196 patches）
- Projector：2-layer MLP（768→1536→1536），4 个可学习 prefix tokens
- LLM：Qwen2.5-1.5B-Instruct，BF16，参数冻结但 forward 保持可导
- 输出：UMS-JSON 格式的结构化医学报告

### 3.2 训练目标

- `L_tok`：Next-token prediction（标准 CE，主监督信号）
- `L_cons`：Augmentation Consistency（cosine similarity，λ=0.1，warmup 2000 步）— 消融证明无正面贡献
- 总 Loss = L_tok + 0.1 × L_cons

### 3.3 V9 原始三组件（消融结论）

| 组件 | 设计意图 | 消融结论 |
| --- | --- | --- |
| GFTM | attention-guided token masking，迫使 ViT 从低注意力区域编码 | 导致预训练过拟合（best@500-1000），去掉后训练稳定且 transfer 更好 |
| FSA Consistency | 冻结 LLM 作为稳定锚点，两次 forward 的 hidden states 做 cosine loss | 去掉后 F1/AUC 均提升，说明约束过强 |
| CMSA | CXR+CT 统一 schema 混合训练（7:3） | 混合 CT 反而拖累 CXR transfer（F1 -0.85pp, AUC -2.36pp） |

**结论**：三个组件均为 net negative，核心价值来自 Frozen LLM Supervision 框架本身。

## 4. V10 新模块：SAR + HFP

基于 V9 消融结论（纯框架最优，GFTM hard masking 有害），V10 提出两个轻量级增强模块：

### 4.1 SAR (Soft Attention Reweighting)

替代 GFTM 的 hard masking，用软加权代替硬丢弃。

- 用 CLS-to-patch attention 的倒数作为软权重，放大低注意力 tokens
- 公式：`token_i = token_i * (1 + α * (w_i - 1))`，其中 `w_i = normalize(1/attn_i)`
- α 为可学习参数（初始值 1.0），训练时自动调节加权强度
- 不丢弃任何 token → 避免 GFTM 的信息损失和过拟合
- 训练和推理都启用，避免分布偏移
- 实现：`models/vit.py` 中 `_apply_sar()` 方法

### 4.2 HFP (Hierarchical Feature Projection)

多层 ViT 特征融合，让 LLM supervision 的梯度传到 ViT 所有层。

- 提取 ViT 第 4/8/12 层（浅层纹理 + 中层结构 + 深层语义）
- 每层一个轻量 MLP projector → concat → fusion layer
- 解决"只有最后一层输出送给 Projector"的信息瓶颈
- 实现：`models/projector.py` 中 `HierarchicalProjector` 类

### 4.3 配置

- `configs/v10_sar_hfp.yaml` — SAR + HFP 联合
- `configs/v10_sar_only.yaml` — SAR 消融
- `configs/v10_hfp_only.yaml` — HFP 消融

## 5. V10.1 SPD 模块

**SPD (Structured Prediction Decomposition)**：多组 cross-attention query tokens 分解视觉表征。

### 5.1 核心思想

VisionProjector 的固定 prefix tokens 不依赖图像内容，对所有图像产生相同的初始表示。SPD 用可学习 query tokens 通过 cross-attention 自适应地从 ViT 特征中提取不同方面的信息。

### 5.2 设计

- 3 组可学习 query tokens（每组 2 个），通过 cross-attention 从 ViT 特征中提取
- 每组独立的 `nn.MultiheadAttention`（4 heads）
- 共享 MLP 投影（vit_dim → llm_dim）
- 正交性损失（ortho_weight=0.01）：鼓励各组 attention pattern 互不重叠

### 5.3 与其他模块的区别

| 模块 | 机制 | 维度 |
| --- | --- | --- |
| VisionProjector | 固定 prefix tokens + MLP | 不依赖图像内容 |
| SAR | 软加权 patch tokens | 单层，token 级别 |
| HFP | 多层 ViT 特征融合 | 纵向多尺度 |
| SPD | cross-attention query 分解 | 横向多方面 |

### 5.4 配置

- `configs/v10.1_spd.yaml`（独立于 SAR/HFP，不叠加）
- 实现：`models/spd.py` 中 `SPDProjector` 类

## 6. 模型配置

| 组件 | 配置 | 参数量 | 状态 |
| --- | --- | --- | --- |
| ViT | vit_base_patch16_224, all tokens | ~86M | 可训练 |
| Projector | 2-layer MLP (768→1536→1536), prefix=4 | ~6M | 可训练 |
| LLM | Qwen2.5-1.5B-Instruct, BF16 | ~1.5B | 冻结 |
| SegHead | 4×ConvTranspose2d (分割 transfer 用) | ~2M | 可训练 |

训练配置：batch=4, grad_accum=4, max_steps=10000, vit_lr=2e-5, proj_lr=1e-4, bf16

## 7. 与 ViTP 差异化

ViTP (Li et al. 2025, arXiv 2509.17562v3) 是最接近的相关工作。

| 维度 | ViTP | VIVID-Med |
| --- | --- | --- |
| Token masking | 随机 drop 75%，固定 | GFTM（已证明有害）→ SAR 软加权 |
| LLM 状态 | 训练 LLM（目标漂移） | 冻结 LLM（稳定锚点） |
| 特征提取 | 单层 | HFP 多层融合 / SPD 多方面分解 |
| 跨模态 | 分开训练 | 消融证明混合无益，不作为 contribution |
| 输出格式 | 自然语言 | 可验证 UMS-JSON |
| 部署形态 | 需要 LLM | ViT-only |
| LLM 规模 | 0.5B | 1.5B |
| 硬件 | 8×A40 | 2×RTX 3090 |

## 8. 已排除的改进方向

| 方向 | 排除原因 |
| --- | --- |
| Uncertainty-Aware Structured Prediction | CheXpert uncertain 标签多为缺失非真正不确定，V3/V4 已尝试 token weighting 效果不好 |
| Attention-Guided Spatial Grounding Loss | CXR 2D 投影，85% 样本多 finding 共存，空间高度重叠，diversity loss 会伤害性能 |
| Adaptive Mask Ratio | 工程复杂度高，novelty 偏弱，固定课程调度已足够 |
| GFTM hard masking | V9 消融证明导致过拟合，已被 SAR 软加权替代 |
| CMSA 跨模态混合训练 | 消融证明混合 CT 拖累 CXR transfer，不作为 contribution |

## 9. 交付物

- **主产物**：medical ViT backbone checkpoint + 训练配方。推理时不需要 LLM。
- **评估维度**：linear probe（主表）+ low-data 曲线 + cross-domain（NIH）+ full fine-tune（补充）+ 分割 transfer（KiTS21）
