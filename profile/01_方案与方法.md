# VIVID-Med V9 SVRL：方案与方法

## 1. 定位

**用冻结 LLM 作为"结构化语义监督空间"，训练 ViT 学到可迁移、可验证的医学视觉表征。训练后丢弃 LLM，只交付 ViT backbone。**

核心叙事（Semantic Knowledge Distillation）：传统 one-hot 标签把疾病当作彼此正交；冻结 LLM 的语义空间天然编码概念相似性与共现拓扑。我们通过结构化 token 对齐训练，让 ViT 表征被蒸馏到 LLM 的语义流形上。

> **LLM 是训练时的老师，ViT 是最终交付物。**

## 2. 三个核心结论 + 三锤硬证据

| 结论 | 硬证据 |
|------|--------|
| LLM 是工具，ViT 是产品 | ViT-only transfer：丢 LLM 后 external/low-data 仍提升 |
| 结构化输出可验证 | 输出约束在 UMS-JSON schema 内 |
| 模型真的看图 | image-swap / prompt-corruption + risk–coverage + 临床硬错误 |

## 3. 核心贡献（V9 SVRL，3 个 novelty）

1. **GFTM (Gradient-Focused Token Masking)**：利用 CLS-to-patch attention 分数优先 mask 高注意力 tokens，迫使 ViT 从低注意力区域编码完整信息。课程学习式 mask_ratio 0.5→0.75。
   - vs ViTP VRL：语义感知 vs 随机；课程递增 vs 固定 75%

2. **FSA (Frozen Semantic Anchor) + Augmentation Consistency**：冻结 LLM 提供稳定语义锚点，对同一图像两次不同 GFTM mask 的 LLM hidden states 做 cosine consistency loss。
   - vs ViTP：ViTP 训练 LLM（目标空间漂移），我们冻结 LLM（目标固定）

3. **CMSA (Cross-Modal Structured Alignment)**：CXR + CT 在统一 UMS-JSON schema 下混合训练（7:3），让 ViT 学到跨模态通用表征。
   - vs ViTP：ViTP 不同领域分开训练，无跨模态对齐

## 4. 方法

### 4.1 前向路径

```
Image → ViT(train) → GFTM(mask) → Projector(train) → Frozen LLM → JSON tokens
                   ↓                                                    ↓
              Second GFTM pass ────────────────────────→ LLM hidden states
                                                                ↓
                                                    Consistency Loss (cosine)
```

### 4.2 关键实现点

- LLM 参数冻结 `requires_grad=False`，但 forward 不能 `no_grad`（需对输入 embedding 可导）
- 训练用 teacher forcing，推理用 constrained JSON decoding
- 输出约束在 UMS-JSON schema 允许的 token 集合中

### 4.3 训练目标

**V9 实际使用：**
- `L_tok`：Next-token prediction（标准 CE，主监督）
- `L_cons`：Augmentation Consistency（cosine similarity，λ=0.1，warmup 2000 步）
- 总 Loss = L_tok + 0.1 × L_cons

**设计蓝图（未实现）：** L_rank（hard-negative ranking）、L_vdep（anti-shortcut）、L_ans（answerability head）

## 5. 模型配置（V9 实际）

| 组件 | 配置 | 参数量 | 状态 |
|------|------|--------|------|
| ViT | vit_base_patch16_224, all tokens | ~86M | 可训练 |
| Projector | 2-layer MLP (768→1536→1536), prefix=4 | ~6M | 可训练 |
| LLM | Qwen2.5-1.5B-Instruct, BF16 | ~1.5B | 冻结 |
| GFTM | attention-guided, mask 0.5→0.75 | — | — |
| SegHead | 4×ConvTranspose2d (分割 transfer 用) | ~2M | 可训练 |

训练配置：batch=4, grad_accum=4, max_steps=10000, vit_lr=2e-5, proj_lr=1e-4, bf16

## 6. 与 ViTP 差异化（审稿人对照表）

| 维度 | ViTP | VIVID-Med V9 |
|------|------|-------------|
| Token masking | 随机 drop 75%，固定 | GFTM：attention-guided，课程递增 |
| LLM 状态 | 训练 LLM（目标漂移） | 冻结 LLM（稳定锚点） |
| Consistency | 无 | FSA：利用冻结 LLM 做 augmentation consistency |
| 跨模态 | 分开训练 | CMSA：CXR+CT 统一 schema 混合训练 |
| 输出格式 | 自然语言 | 可验证 UMS-JSON |
| 部署形态 | 需要 LLM | ViT-only |

**撞车风险：中低。** 同属"用语言模型监督视觉骨干"大类，但 GFTM/FSA/CMSA 三个 novelty 各有明确消融支撑，输出形态和评估维度完全不同。

## 7. 交付物

- **主产物**：medical ViT backbone checkpoint + 训练配方。推理时不需要 LLM。
- **评估维度**：分类 transfer（CheXpert→NIH）+ 分割 transfer（KiTS21 + low-data）+ 消融

## 8. 待探索方向：Multi-Scale Token Aggregation (MSTA)

### 8.1 动机

当前 ViT 只取最后一层输出送入 Projector。浅层（layer 4）编码纹理/边缘（如骨折线、气胸边界），深层（layer 12）编码语义。只用最后一层会丢失浅层细粒度信息，对 Pneumothorax、Fracture 等依赖局部纹理的标签不利。

### 8.2 方案

从 ViT 第 4/8/12 层取 multi-scale features，用 cross-attention 聚合后送入 Projector：

```
Layer 4 tokens ──┐
Layer 8 tokens ──┼── Cross-Attention Aggregator ── Projector ── LLM
Layer 12 tokens ─┘
```

- 参数量增加很少（一个小的 cross-attention module）
- ViTP 没有做 multi-scale，这是差异化点
- 预期对 Pneumothorax（F1 0.426，当前最弱项）和 Fracture 有帮助

### 8.3 状态

- 尚未实现，作为 P2 加分项
- 需要先完成 P0 消融实验，确认当前 3 个 novelty 各自贡献后再决定是否加入

## 9. 已排除的改进方向（附原因）

### 9.1 Uncertainty-Aware Structured Prediction ❌

- CheXpert 的 uncertain 标签（-1）多为缺失/不可答，不是真正的认知不确定性
- V3/V4 已尝试 token weighting（present:3.0, uncertain:2.5, absent:1.5），效果不好
- 数据统计：Atelectasis 3630 uncertain, Consolidation 2833, Pneumonia 2092
- 结论：这条路在 V3/V4 已经走过，不值得再投入

### 9.2 Attention-Guided Spatial Grounding Loss ❌

- CXR 是 2D 投影，不同 finding 的空间区域高度重叠
- 数据统计：85% 样本有 2+ findings 共存
- 强共现对：Lung Opacity+Support Devices 30%, Lung Opacity+Edema 22%, Edema+Support Devices 24%
- 硬做 diversity loss 会伤害性能（多个 finding 本来就该关注同一区域）
- 结论：CXR 的空间特性不适合做 finding 级别的互斥 grounding

### 9.3 Adaptive Mask Ratio ❌

- 根据 loss 趋势动态调整 mask_ratio
- 问题：不同 batch 的 loss 波动大，需要滑动平均/趋势检测，设上下限防止跑到极端
- 工程复杂度高，novelty 偏弱（更像 trick 而非方法贡献）
- 结论：当前固定课程调度（0.5→0.75）已足够，不值得增加复杂度
