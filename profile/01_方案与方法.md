# VIVID-Med：方案与方法

> 最后更新：2026-02-21（V12 重构版，SPD only，全部实验已完成）

## 1. 定位

**用冻结 LLM 作为"结构化语义监督空间"，训练 ViT 学到可迁移、可验证的医学视觉表征。训练后丢弃 LLM，只交付 ViT backbone。**

核心叙事（Semantic Knowledge Distillation）：传统 one-hot 标签把疾病当作彼此正交；冻结 LLM 的语义空间天然编码概念相似性与共现拓扑。我们通过结构化 token 对齐训练，让 ViT 表征被蒸馏到 LLM 的语义流形上。

> **LLM 是训练时的老师，ViT 是最终交付物。**

## 2. 核心方法（SPD + 12-label）

主方法 = Frozen LLM Supervision + UMS 结构化 JSON + SPD projector + 12-label 预训练。

### 2.1 架构总览

```text
Image → ViT(trainable) → vit_features (CLS + 196 patches)
      → SPD Projector:
         多组 query tokens × cross-attention on vit_features → SPD tokens
         共享 MLP 投影: SPD tokens + patch tokens → LLM embedding space
      → Frozen LLM (Qwen2.5-1.5B-Instruct) → UMS-JSON token logits
```

### 2.2 SPD (Structured Prediction Decomposition)

多组 cross-attention query tokens 分解视觉表征。

- 3 组可学习 query tokens（每组 2 个 = 6 SPD tokens），通过 cross-attention 从 ViT 特征中自适应提取不同方面信息
- 每组独立的 `nn.MultiheadAttention`（4 heads）
- 正交性损失（ortho_weight=0.01）：鼓励各组 attention pattern 互不重叠
- 共享 MLP 投影（vit_dim → llm_dim）
- 与 VisionProjector 的区别：prefix tokens 不依赖图像内容 → SPD query tokens 自适应提取
- 实现：`models/spd.py` 中 `SPDProjector` 类

### 2.3 训练策略

**Field Query Training**：每样本随机抽 4-6 个标签生成 JSON，重点抽低频标签（Fracture/Pneumonia/Lung Lesion 等，focus_probability=0.6）。增加训练多样性，缓解高频标签主导。

**Answerability Mask**：对 state=null（不可回答）的 findings，其 token loss 权重设为 0（false_weight=0.0）。只在可回答的 findings 上计算 loss，避免 null state 的噪声梯度。

### 2.4 训练目标

- `L_tok`：Next-token prediction（标准 CE，主监督信号，仅在 answerable findings 上计算）
- `L_ortho`：SPD 正交性损失（ortho_weight=0.01）
- 总 Loss = L_tok + 0.01 × L_ortho

### 2.5 12-label 预训练口径

去掉 No Finding（全阳性，AUC=NaN）和 Pleural Other（极低频 <50 样本，AUC 不稳定）。12 个标签：Enlarged Cardiomediastinum, Cardiomegaly, Lung Opacity, Lung Lesion, Edema, Consolidation, Pneumonia, Atelectasis, Pneumothorax, Pleural Effusion, Fracture, Support Devices。

## 3. 模型配置

| 组件 | 配置 | 参数量 | 状态 |
| --- | --- | --- | --- |
| ViT | vit_base_patch16_224, all tokens | ~86M | 可训练 |
| SPD Projector | 3组×2tokens + cross-attention + 共享MLP | ~6M | 可训练 |
| LLM | Qwen2.5-1.5B-Instruct, BF16 | ~1.5B | 冻结 |

训练配置：batch=4, grad_accum=8, max_steps=10000, vit_lr=2e-5, proj_lr=1e-4, bf16

## 4. 与 ViTP 差异化

ViTP (Li et al. 2025, arXiv 2509.17562v3) 是最接近的相关工作。

| 维度 | ViTP | VIVID-Med |
| --- | --- | --- |
| LLM 状态 | 训练 LLM（目标漂移） | 冻结 LLM（稳定锚点） |
| Token 处理 | 随机 drop 75%，固定 | SPD cross-attention 分解（不丢弃 token） |
| Projector | 简单 MLP | SPD 多组 query + 正交性约束 |
| 输出格式 | 自然语言 | 可验证 UMS-JSON |
| 部署形态 | 需要 LLM | ViT-only |
| LLM 规模 | 0.5B | 1.5B |
| 硬件 | 8×A40 | 2×RTX 3090 |

## 5. 已排除的设计

| 方向 | 排除原因 |
| --- | --- |
| GFTM hard masking | V9 消融证明导致过拟合（best@500-1000 后退化） |
| FSA Consistency loss | V9 消融证明无正面贡献 |
| CMSA 跨模态混合训练 | 消融证明混合 CT 拖累 CXR transfer |
| SAR 软加权 | V10 实验表明增益有限，已被 SPD 替代 |
| HFP 多层融合 | V12 实验表明 SPD+HFP 联合反而低于 SPD 单独（LP AUC 0.7930 vs 0.8441） |
| Label-Balanced Loss | V11 实验表明逆频率加权扭曲 loss landscape |

## 6. 交付物

- **主产物**：medical ViT backbone checkpoint + 训练配方。推理时不需要 LLM。
- **评估维度**：linear probe（主表）+ cross-domain（NIH）+ 消融递进链
