## 2. Introduction 三句话模板

1. 现有医学视觉训练多依赖 one-hot/弱标签监督，难以编码概念相似性与共现结构。
2. 我们提出将冻结 LLM 视为语义监督空间，通过结构化 JSON token 预测训练 ViT 表征，训练后丢弃 LLM 只保留 ViT。
3. SPD（Structured Prediction Decomposition）通过多组正交 query tokens 分解视觉表征，在 LP 和跨域评估下验证表征质量。

## 3. Contribution Bullets（英文，放 Intro 末尾）

- **Frozen LLM Supervision Framework.** We propose using a frozen LLM as a structured semantic supervision space for training visual representations. The LLM is discarded after training, yielding a deployable ViT-only backbone.
- **Unified Medical Schema (UMS).** We design a structured JSON supervision format that encodes findings as verifiable field-state pairs, enabling richer gradient signals compared to free-text supervision.
- **Structured Prediction Decomposition (SPD).** We introduce learnable query token groups with cross-attention and orthogonality regularization, enabling the projector to decompose structured predictions into complementary aspects and provide richer gradient signals to the ViT encoder.
- **Comprehensive Evaluation.** We validate representation quality through linear probing on CheXpert (12-label), cross-domain transfer to NIH ChestX-ray14, and a single-variable ablation chain (free-text → UMS → UMS+SPD) that isolates each component's contribution.

## 4. Related Work 定位段（英文 paper-ready）

### Visual instruction pretraining

Recent works use language supervision to improve visual representations. A representative line embeds a vision encoder into a VLM and optimizes next-token prediction on instruction data, then extracts the pretrained backbone for downstream tasks. While effective, these approaches rely on free-form textual responses and often target improving the overall VLM.

### Medical foundation models

Medical foundation models are commonly learned via self-supervised objectives (MAE, DINO) or contrastive learning (BiomedCLIP). However, clinical deployment prioritizes robustness under domain shift, calibration, and avoidance of clinically critical errors, which are not explicitly controlled by most generic pretraining objectives.

### Our positioning

We differ in both goal and supervision design. Our goal is a deployable ViT-only medical backbone, not an instruction-following VLM. We treat a frozen LLM as a semantic teacher and update only ViT+projector via structured, verifiable token supervision. SPD decomposes visual features into orthogonal aspects for richer supervision signals.

> **One-sentence contrast to ViTP:** Unlike visual instruction pretraining that learns from free-form responses with a trainable LLM, we perform verifiable semantic distillation where a frozen LLM provides structured supervision to train a deployable ViT-only model.

### 7.3 与 ViTP 的差异化叙事

| 维度 | ViTP | 我们 |
| --- | --- | --- |
| LLM 角色 | 训练 LLM（目标空间漂移） | 冻结 LLM（稳定语义锚点） |
| Token 处理 | 随机 drop 75% | SPD cross-attention 分解（不丢弃） |
| Projector | 简单 MLP | SPD 多组 query + 正交性约束 |
| 输出格式 | 自然语言 | 可验证 UMS-JSON |
| 部署 | 需要 LLM | ViT-only |

### 7.4 写作注意事项

- ViTP 无法直接对比（无开源权重），用 random mask 消融作为 proxy
- 不要在 Related Work 中过度攻击 ViTP，定位为"同属大类，不同分叉"
- 强调冻结 LLM 是优势（稳定锚点）而非限制
- 主表口径用 LP（表征质量差异最大化）
- 指标口径：12-label F1 + 12-label AUC

## 8. 常见问题速查（QA）

**Q: 最终交付物是什么？**
A: 医学 ViT backbone checkpoint。部署时不需要 LLM，直接接下游 head。

**Q: 为什么用 LP 而不是 FT？**
A: LP 是表征学习标准评估（SimCLR/DINO/MAE/ViTP 均采用），FT 压缩差异。

**Q: 为什么报 12-label？**
A: No Finding（全阳性，AUC=NaN）和 Pleural Other（<50 样本）导致 macro 指标不稳定。12-label 口径在 CheXpert 论文中常见。

**Q: SPD 的 novelty 在哪？**
A: 现有 VLP projector 多为简单 MLP 或 Q-Former（面向 VLM 推理），SPD 专为表征学习设计：多组 query tokens 通过正交性约束分解不同语义方面，为 ViT 提供更丰富的梯度信号。

**Q: 为什么不用 HFP？**
A: 历史实验表明 SPD+HFP 联合反而低于 SPD 单独（LP AUC 0.7930 vs 0.8441）。SPD 的 cross-attention 已能提供足够的梯度信号，HFP 叠加增加复杂度但干扰训练。

**Q: 和 ViTP 是不是一类？**
A: 同属"用语言信号塑造视觉骨干"大类，但关键分叉：(1) 冻结 vs 训练 LLM，(2) SPD 结构化分解 vs 随机 mask，(3) 可验证 JSON vs 自然语言。
