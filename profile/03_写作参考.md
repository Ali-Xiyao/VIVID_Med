## 2. Introduction 三句话模板

1. 现有医学视觉训练多依赖 one-hot/弱标签监督，难以编码概念相似性与共现结构，在跨域与干扰下容易出现 shortcut。
2. 我们提出 SVRL：将冻结 LLM 视为语义监督空间，通过结构化 JSON token 预测训练 ViT 表征。
3. 训练后丢弃 LLM，仅保留 ViT，在 external/low-data/可靠性指标上验证价值。

> 类比：**老师（LLM）只在训练时教，毕业后上岗的是学生（ViT）**。

## 3. Contribution Bullets（英文，放 Intro 末尾）

- **GFTM.** Attention-guided token masking with curriculum scheduling.
- **FSA + Consistency.** Frozen LLM as stable semantic anchor with augmentation consistency.
- **CMSA.** Cross-modal (CXR+CT) unified training under verifiable structured schema.
- **Evaluation.** ViT-only transfer under domain shift, low-data, and reliability metrics.

## 4. Related Work 定位段（英文 paper-ready）

### Visual instruction pretraining

Recent works use language supervision to improve visual representations. A representative
line embeds a vision encoder into a VLM and optimizes next-token prediction on instruction
data, then extracts the pretrained backbone for downstream tasks. While effective, these
approaches rely on free-form textual responses and often target improving the overall VLM.

### Medical foundation models

Medical foundation models are commonly learned via self-supervised objectives (MAE, DINO)
or contrastive learning (BiomedCLIP). However, clinical deployment prioritizes robustness
under domain shift, calibration, and avoidance of clinically critical errors, which are not
explicitly controlled by most generic pretraining objectives.

### Our positioning

We differ in both goal and supervision design. Our goal is a deployable ViT-only medical
backbone, not an instruction-following VLM. We treat a frozen LLM as a semantic teacher
and update only ViT+projector via structured, verifiable token supervision.

> **One-sentence contrast to ViTP:** Unlike visual instruction pretraining that learns from
> free-form responses, we perform verifiable semantic distillation where a frozen LLM provides
> structured supervision to train a deployable ViT-only model.

## 7. 常见问题速查（QA）

**Q: 最终交付物是什么？**
A: 医学 ViT backbone checkpoint。部署时不需要 LLM，直接接下游 head。

**Q: ViT-only transfer 为什么关键？**
A: 直接回答"提升来自 LLM 兜底还是 ViT 表征变强"。

**Q: 和 ViTP 是不是一类？**
A: 同属"用语言信号塑造视觉骨干"大类，但三个硬分叉：(1) GFTM vs 随机 mask，(2) 冻结 LLM + consistency vs 训练 LLM，(3) 跨模态统一 schema vs 分开训练。

**Q: novelty 会不会偏弱？**
A: 3 个 novelty 各有消融支撑。关键是至少 1 个带来 >1% macro_F1 提升，且在 low-data/跨域场景优势更明显。

**Q: 这是不是多任务学习？**
A: 不是。我们把不同监督统一到 UMS-JSON 的结构化 token 预测接口（单一输出空间），而非多个任务头。

**Q: 如果比不过 BiomedCLIP？**
A: 不硬拼通用 VLM 能力，主战场放在 ViT-only + external/low-data/可靠性。可加 BiomedCLIP init + Ours 证明贡献是训练机制。

**Q: 必须和 task-head baseline 对比吗？**
A: 必须，放主表第一行。不比容易被判 busywork。即使同分布不赢，external/low-data/可靠性赢就够。
