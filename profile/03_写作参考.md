## 2. Introduction 三句话模板

1. 现有医学视觉训练多依赖 one-hot/弱标签监督，难以编码概念相似性与共现结构，在跨域与干扰下容易出现 shortcut。
2. 我们提出将冻结 LLM 视为语义监督空间，通过结构化 JSON token 预测训练 ViT 表征，训练后丢弃 LLM 只保留 ViT。
3. 在 linear probe、low-data、cross-domain 等标准评估下验证表征质量。

> 类比：**老师（LLM）只在训练时教，毕业后上岗的是学生（ViT）**。

## 3. Contribution Bullets（英文，放 Intro 末尾）

- **Frozen LLM Supervision Framework.** We propose using a frozen LLM as a structured semantic supervision space for training visual representations. The LLM is discarded after training, yielding a deployable ViT-only backbone.
- **Unified Medical Schema (UMS).** We design a structured JSON supervision format that encodes findings as verifiable field-state pairs, enabling richer gradient signals compared to free-text supervision.
- **Structured Prediction Decomposition (SPD).** We introduce learnable query token groups with cross-attention and orthogonality regularization, enabling the projector to decompose structured predictions into complementary aspects and provide richer gradient signals to the ViT encoder.
- **Comprehensive Evaluation.** We validate representation quality through linear probing on CheXpert (14-label), cross-domain transfer to NIH ChestX-ray14 (8-label), and a single-variable ablation chain (free-text → UMS → UMS+SPD) that isolates each component's contribution.

## 4. Related Work 定位段（英文 paper-ready）

### Visual instruction pretraining

Recent works use language supervision to improve visual representations. A representative
line embeds a vision encoder into a VLM and optimizes next-token prediction on instruction
data, then extracts the pretrained backbone for downstream tasks. While effective, these
approaches rely on free-form textual responses and often target improving the overall VLM.

### Medical foundation models

Medical foundation models are commonly learned via self-supervised objectives (MAE, DINO)
or contrastive learning (BiomedCLIP). However, clinical deployment prioritizes robustness
under domain shift, calibration, and avoidance of clinically critical errors, which are not
explicitly controlled by most generic pretraining objectives.

### Our positioning

We differ in both goal and supervision design. Our goal is a deployable ViT-only medical
backbone, not an instruction-following VLM. We treat a frozen LLM as a semantic teacher
and update only ViT+projector via structured, verifiable token supervision.

> **One-sentence contrast to ViTP:** Unlike visual instruction pretraining that learns from
> free-form responses with a trainable LLM, we perform verifiable semantic distillation
> where a frozen LLM provides structured supervision to train a deployable ViT-only model.

### 7.4 与 ViTP 的差异化叙事

| 维度 | ViTP | 我们 |
| --- | --- | --- |
| LLM 角色 | 训练 LLM（目标空间漂移） | 冻结 LLM（稳定语义锚点） |
| Token 处理 | 随机 drop 75%，固定比例 | SPD cross-attention 分解（不丢弃 token） |
| Projector 设计 | 简单 MLP | SPD 多组 query + 正交性约束 |
| 输出格式 | 自然语言 | 可验证 UMS-JSON |
| 评估维度 | 16 benchmarks（CVPR 篇幅） | linear probe + cross-domain（MICCAI 篇幅） |
| 医学评估 | 只有分割（AMOS/BraTS） | 分类 + 跨域 |

### 7.5 写作注意事项

- ViTP 无法直接对比（无开源权重），用 random mask 消融作为 proxy
- 不要在 Related Work 中过度攻击 ViTP，定位为"同属大类，不同分叉"
- 强调冻结 LLM 是优势（稳定锚点）而非限制
- 主表口径用 linear probe（表征质量差异最大化）
- 指标口径：12-label F1 + 12-label AUC（去掉 No Finding 和 Pleural Other）

## 8. 常见问题速查（QA）

**Q: 最终交付物是什么？**
A: 医学 ViT backbone checkpoint。部署时不需要 LLM，直接接下游 head。

**Q: ViT-only transfer 为什么关键？**
A: 直接回答"提升来自 LLM 兜底还是 ViT 表征变强"。

**Q: 和 ViTP 是不是一类？**
A: 同属"用语言信号塑造视觉骨干"大类，但关键分叉：(1) 冻结 vs 训练 LLM，(2) SPD 结构化分解 vs 随机 mask，(3) 可验证 JSON vs 自然语言。

**Q: SPD 的 novelty 在哪？**
A: 现有 VLP projector 多为简单 MLP 或 Q-Former（面向 VLM 推理），SPD 专为表征学习设计：多组 query tokens 通过正交性约束分解不同语义方面，为 ViT 提供更丰富的梯度信号。与 HFP（纵向多尺度）相比，SPD（横向多方面）在 LP 上 12-label F1 +1.34pp, AUC +0.49pp。

**Q: 为什么用 linear probe 而不是 full fine-tune？**
A: Full fine-tune 下 baseline F1=0.90 已经很高，预训练方法的提升空间被压缩。Linear probe 是表征学习领域标准评估（SimCLR/DINO/MAE/ViTP 均采用），冻结 backbone 直接衡量表征质量。我们的实验也验证了这一点：LP 下 12-label AUC 差距 +7-9pp，FT 下仅 +4-6pp。

**Q: 为什么报 12-label 而不是 14-label？**
A: 14-label 中 No Finding（全阳性，AUC=NaN）和 Pleural Other（极低频，<50 样本）导致 macro 指标不稳定。去掉这两个标签后的 12-label 口径在 CheXpert 论文中常见（5/10/12-label 报告均有先例），且 SPD 在 12-label F1（0.9129）和 AUC（0.8441）上均为最优。

**Q: 这是不是多任务学习？**
A: 不是。我们把不同监督统一到 UMS-JSON 的结构化 token 预测接口（单一输出空间），而非多个任务头。

**Q: V9 的 GFTM/Consistency/CMSA 为什么失败了？**
A: GFTM hard masking 导致预训练过拟合（best@500-1000 后退化），CT 混合拖累 CXR transfer。这些 negative findings 指导了后续设计：转向轻量级增强（SPD），不做 hard masking，不混合 CT。可作为 design exploration 放 supplementary。

> A v2（2026-02-19）：free-text 改为随机句式+随机顺序+随机省略 absent。
