# 方法（最终版）：冻结 LLM 的结构化对齐训练

## 0. 训练对象与输入/输出定义（写清楚就不会显得“在玩 LLM”）

一个训练样本在进入模型前，先被编译为 **UMS-JSON**（结构化、可验证、允许 `null`）：

- 图像 `x`：CXR 单张；CT 用 2.5D（v1.0 默认：相邻 **3** 切片拼成 3 通道输入）
- 结构化目标 `y`：UMS-JSON（序列化为 token 序列 `y1…yT`）
- 字段级可答性 `a`：`answerability.<field> ∈ {true,false}`（不可答字段目标固定为 `null/uncertain`）
- verifier 输出 `r`：`role ∈ {positive,abstain,negative-only,drop}`（决定进入哪些 loss）
- （可选）hard negatives `y^-`：满足 schema 但内容错误的 JSON（laterality/unit/×10/confusable findings 等）

模型训练时的 **输入/输出**：

- 输入：`(x, prompt)`（prompt 可固定模板；训练用 teacher forcing 还会喂入 `y_<t`）
- 输出：对每个 time step 的 **next-token logits**（用于预测结构化 JSON tokens），以及（可选）字段级 answerability 预测

> 关键点：我们把“分类/分割/回归”等不同监督，统一成一个 **结构化 token 预测接口**；LLM 不是推理时的聊天引擎，而是训练期的“语义监督空间”。

## 1. 模型组件

- **Vision Encoder（ViT，可训练）**：输出视觉 token/features。
- **Projector（可训练）**：把视觉 token 映射到 LLM hidden dim，并作为“视觉前缀 tokens”拼接到文本前。
  - v1.0 推荐：**MLP + learnable prefix tokens（16 或 32）**
- **Frozen LLM（参数冻结）**：作为结构化语义解码器/监督空间，产生 JSON token 的 logits。

> 目标不是提升“多模态聊天能力”，而是用 LLM 作为监督接口来训练 ViT 表征。

## 2. 前向路径（必须写清楚，避免 technical incorrect）

**Image → ViT(train) → Projector(train) → Frozen LLM(forward, params frozen) → JSON token logits**

### 2.1 冻结参数 ≠ no_grad（命门）

- 冻结参数：`for p in llm.parameters(): p.requires_grad = False`
- 但 **不能** 用 `torch.no_grad()` 包住 LLM forward
  否则梯度无法回传到 projector/ViT，ViT 学不到任何东西。

论文一句话写法（建议直接用）：

> We freeze LLM parameters while keeping the forward pass differentiable w.r.t. the input embeddings, enabling gradients to update only the vision encoder and projector.

## 3. 结构化受约束生成（Constrained JSON）

我们不做自由文本生成；输出被限制在 **UMS-JSON schema** 允许的 token 集合中。

### 3.1 训练时（Training）

- v1.0 推荐：**Constraint-aware Teacher Forcing（masked-CE，不做解码搜索）**训练仍然采用 teacher forcing（喂真值 token 序列），但在每个 step `t`：
  1. 根据 JSON grammar + 前缀 `y_<t` 计算合法 token 集 `A_t`
  2. 将非法 token 的 logits 置为 `-∞`
  3. 再做 softmax + cross entropy

这样训练阶段的 token 支持集与推理阶段 constrained decoding **严格一致**，但成本接近普通 CE（不需要 beam/search）。

### 3.2 推理/评测时（Inference/Eval）

- 使用 constrained decoding（同一套 grammar/FSM/trie 与训练时一致）。
- constrained decoding 失败必须进入 failure taxonomy（schema_violation 等），并计入可靠性统计。

## 4. Hard negatives（格式合法但内容错误）

每条样本构造 3–5 个 hard negative JSON（均满足 schema）：

- laterality flip：left ↔ right
- unit swap：mm ↔ cm
- measurement ×10 / ÷10
- confusable finding swap（需要维护“可混淆对”字典）

训练可选两种实现（二选一即可）：

1. **sequence-level margin ranking**：`score(y_true) > score(y_neg) + m`
2. **DPO/IPO 风格偏好损失**：只更新 ViT+projector（LLM 冻结）

## 5. anti-shortcut 机制（训练 + 硬评测）

- **prompt corruption**：同语义不同措辞 → 输出应保持一致
- **image swap**：同 prompt 换图 → 输出必须变化（尤其 findings/measurements）

这些扰动既用于训练正则（见 `06_训练目标与SamplePolicy.md`），也用于硬评测（见 `07_实验矩阵与评估协议.md`）。

## 6. 部署/使用形态（论文里怎么讲“可用且有用”）

> 医学论文里最稳的口径：**训练期可以重，部署期必须轻**。

- **形态 A（主线，默认）**：ViT-only  
  训练后丢弃 LLM，只保留 ViT backbone；在下游任务（分类/分割/回归/检索）上接轻量 head（或下游框架）做微调/线性探测，强调 external/low-data/可靠性收益。
- **形态 B（可选加分）**：ViT + Frozen LLM（结构化报告/交互层）
  作为"可选交互层"做结构化输出（UMS-JSON），但不把它当主交付物；主结论仍以 ViT-only 为准。

## 7. V9 SVRL 新增方法组件（2026-02-13）

V9 在原有 LLM-mediated Structured Alignment 基础上，新增三个训练机制：

### 7.1 GFTM (Gradient-Focused Token Masking)

**动机**：即使使用 all-tokens 输出，Projector 仍可能"代劳"大部分对齐工作，ViT 表征改变不足。通过 mask 掉 ViT 已经关注的 tokens，迫使 ViT 从低注意力区域也编码完整信息。

**实现**：
1. 正常 forward 得到 ViT features (B, 197, 768)
2. 通过 attention hook 捕获最后一个 block 的 CLS-to-patch attention weights
3. 按 attention 分数排序，保留低注意力 tokens（mask 高注意力 tokens）
4. mask_ratio 从 0.5 线性增到 0.75（课程学习式难度递增）

**与 ViTP VRL 的关键区别**：
- VRL：随机 drop 75% tokens，固定比例
- GFTM：语义感知（基于 attention），课程学习式递增，优先 mask "已学会"的区域

**论文一句话**：
> We propose Gradient-Focused Token Masking (GFTM), which leverages CLS-to-patch attention scores to preferentially mask high-attention tokens, forcing the ViT to encode complete information from under-attended regions.

### 7.2 FSA (Frozen Semantic Anchor) + Augmentation Consistency

**动机**：冻结 LLM 提供稳定的语义锚点空间（不像 ViTP 训练 LLM 导致目标空间漂移）。对同一图像的两次不同 GFTM mask 的 forward pass，经过 ViT→Projector→LLM 后的 hidden states 应该一致。

**实现**：
1. 对同一 batch 做两次 forward（不同 GFTM mask）
2. 取两次 LLM hidden states 的 mean pooling
3. 计算 cosine similarity loss：`L_cons = 1 - cos(h1, h2)`
4. 总 loss = L_lm + λ_cons × L_cons（λ_cons=0.1，warmup 2000 步后启用）

**论文叙事**：
> The frozen LLM serves as a stable semantic anchor: since its parameters never change, the consistency loss has a well-defined optimization target. This transforms the "frozen LLM" constraint from a limitation into an architectural advantage.

### 7.3 CMSA (Cross-Modal Structured Alignment)

**动机**：单一模态（CXR）限制了训练数据量和表征泛化性。通过统一 UMS-JSON schema，可以在同一个训练框架下混合 CXR 和 CT 数据。

**实现**：
1. AMOS 2.5D CT 数据（26k slices）+ CheXpert CXR 数据（30k samples）
2. `MultiModalSampler` 按 CXR:CT = 7:3 比例采样
3. 不同模态的 UMS-JSON 共享相同的顶层 schema（modality, anatomy, findings 等字段）
4. 标签维度不同（CXR 7 vs CT 15）通过 NaN padding 对齐

**与 ViTP 的区别**：
- ViTP：不同领域（remote sensing, medical）分开训练，没有跨模态对齐
- CMSA：在统一 schema 下混合训练，让 ViT 学到跨模态通用的医学视觉表征

### 7.4 V9 完整前向路径

```
Image → ViT(train) → GFTM(mask low-info tokens) → Projector(train) → Frozen LLM → JSON tokens
                   ↓                                                              ↓
              Second GFTM pass ──────────────────────────────────→ LLM hidden states
                                                                          ↓
                                                              Consistency Loss (cosine)
```

### 7.5 分割 Transfer 评估（新增）

除分类 transfer 外，V9 新增分割 transfer 评估：
- 提取 ViT backbone（丢弃 Projector + LLM）
- 接 `ViTSegHead`：patch tokens → 4 层 ConvTranspose2d 上采样 → (B, num_classes, 224, 224)
- 在 KiTS21 上 fine-tune + 评估（Dice score）
- Low-data 曲线（10%, 25%, 50%, 100%）证明低数据场景优势
