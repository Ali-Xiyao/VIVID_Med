# VIVID-Med 导师汇报

> 最后更新：2026-02-19

## 一句话概述

用冻结 LLM 作为结构化语义监督空间训练 ViT，训练后丢弃 LLM，只交付 ViT backbone。在 CheXpert linear probe 和 NIH 跨域评估上均显著超越 ImageNet baseline。

## 1. 论文主实验（已完成）

### 1.1 主表：Linear Probe（⭐ 冻结 ViT，只训练线性分类头）

评估协议：CheXpert 14-label 全量（~29k 训练样本），冻结 ViT backbone，只训练线性分类头。Checkpoint 选择规则：取 val_loss 最低的 step 报告所有指标。指标口径：12-label（去掉 No Finding 和 Pleural Other）。

| 实验 | best step | val_loss | 12-label F1 | 12-label AUC | vs Baseline (F1 / AUC) |
| --- | ---: | ---: | ---: | ---: | --- |
| Baseline (ImageNet) | 2000 | 0.3047 | 0.8845 | 0.7523 | — |
| Random mask (VRL 风格) | 3000 | 0.2913 | 0.8603 | 0.7997 | -2.42pp / +4.74pp |
| No mask+No cons（纯框架） | 2800 | 0.2584 | 0.8939 | 0.8296 | +0.94pp / +7.73pp |
| V10 SAR+HFP | 2600 | 0.2520 | 0.8995 | 0.8392 | +1.50pp / +8.69pp |
| **V10.1 SPD** | 2600 | **0.2461** | **0.9129** | **0.8441** | **+2.84pp / +9.18pp** |

> 12-label = 去掉 No Finding（全阳性，AUC=NaN）和 Pleural Other（极低频，<50 样本，AUC 不稳定）。CheXpert 论文常见 5/10/12-label 报告口径。SPD 在 12-label F1 和 AUC 上均为最优。
>
> Random mask F1 低于 baseline（0.8603 vs 0.8845），主因 Pneumothorax F1=0.2469（SPD 为 0.6235）。VRL 风格 75% random masking 在小数据量下损害表征质量。

配置文件：
- Baseline: `configs/linear_probe_imagenet_full14.yaml`
- Random mask: `configs/linear_probe_random_mask_full14.yaml`
- No mask+No cons: `configs/linear_probe_no_mask_no_cons_full14.yaml`
- V10 SAR+HFP: `configs/linear_probe_v10_sar_hfp_full14.yaml`
- V10.1 SPD: `configs/linear_probe_v10.1_spd_full14.yaml`

Linear probe 超参数：backbone_lr=0, head_lr=1e-3, weight_decay=0, max_steps=3000, eval_interval=200, drop_path_rate=0.0, batch_size=16, grad_accum=2, bf16=true。

### 1.2 NIH 跨域泛化（8 共同标签，25,596 样本）

用 CheXpert 训练的 linear probe checkpoint 直接在 NIH ChestX-ray14 上测试，不重新训练。

| 实验 | macro_F1 | macro_AUC | vs Baseline |
| --- | ---: | ---: | --- |
| Baseline (ImageNet) | 0.2033 | 0.6159 | — |
| No mask+No cons | 0.2384 | 0.6748 | F1 +3.51pp, AUC +5.89pp |

### 2.2 V9 设计探索（Supplementary / Appendix）

V9 SVRL 原始消融（GFTM + Consistency + CT 混合），结论：三个组件均无正面贡献，核心价值来自 frozen LLM supervision 框架本身。此结论指导了 V10/V10.1/V11 的设计方向。

| 实验 | GFTM | Cons | CT | macro_F1 (FT) | macro_AUC (FT) |
| --- | --- | --- | --- | ---: | ---: |
| SVRL-full | ✓ | ✓ | ✓ | 0.9064 | 0.8210 |

## 3. V10/V10.1 新模块（基于消融结论设计）

消融结论：纯框架最优，GFTM hard masking 有害。V10/V10.1 在纯框架基础上加轻量级增强模块。

### 3.1 V10 SAR+HFP

- **SAR (Soft Attention Reweighting)**：用 CLS-to-patch attention 倒数作为软权重，放大低注意力 tokens，替代 GFTM 的 hard masking。可学习参数 α 控制加权强度。
- **HFP (Hierarchical Feature Projection)**：提取 ViT 第 4/8/12 层特征，每层一个 MLP projector → concat → fusion，让梯度传到 ViT 所有层。
- 预训练配置：`configs/v10_sar_hfp.yaml`（10k steps, CXR+CT）
- Linear probe 结果：12-label F1=0.8995, AUC=0.8392（vs Baseline +1.50pp/+8.69pp）

### 3.2 V10.1 SPD（当前最优）

- **SPD (Structured Prediction Decomposition)**：3 组可学习 query tokens（每组 2 个），通过 cross-attention 从 ViT 特征中自适应提取不同方面信息。正交性损失（ortho_weight=0.01）鼓励各组 attention pattern 互不重叠。
- 与 HFP 的区别：HFP 纵向多尺度（多层融合）→ SPD 横向多方面（单层分解）
- 预训练配置：`configs/v10.1_spd.yaml`（10k steps, CXR+CT）
- Linear probe 结果：**12-label F1=0.9129**（所有实验最高），**AUC=0.8441**（所有实验最高）
- NIH 跨域：**F1=0.2599, AUC=0.7121**（所有实验最高）

## 7. 论文故事线（Paper Story）

### 7.1 一句话卖点

现有医学视觉预训练要么依赖 one-hot 标签（丢失语义关系），要么训练整个 VLM（部署成本高）。我们提出一种新范式：**用冻结 LLM 作为结构化语义老师训练 ViT，毕业后只部署 ViT**。

### 7.2 故事三幕结构

**第一幕：问题（Why）**

医学影像分类的标准做法是 ImageNet 预训练 → 下游 fine-tune。但 one-hot 标签有两个根本缺陷：
1. 语义扁平：Cardiomegaly 和 Pleural Effusion 在 one-hot 空间正交，但临床上高度共现（心衰患者常同时出现）
2. 跨域脆弱：one-hot 学到的表征依赖训练集的标签分布，换一个数据集（CheXpert → NIH）性能大幅下降

近期 ViTP 等工作尝试用 LLM 监督训练视觉模型，但它们训练 LLM 本身（目标空间漂移）、使用自由文本输出（不可验证）、随机丢弃 75% token（信息损失）。

**第二幕：方法（What）**

我们的核心洞察：**LLM 的语义空间天然编码了疾病间的概念相似性和共现关系**。不需要训练 LLM，只需要把它当作一个固定的语义坐标系，让 ViT 学会把图像映射到这个坐标系上。

三个技术贡献：
1. **Frozen LLM Supervision**：冻结 Qwen2.5-1.5B 作为稳定语义锚点，ViT+Projector 通过预测结构化 JSON tokens 学习表征。训练后丢弃 LLM，只交付 ViT。
2. **UMS（Unified Medical Schema）**：将医学发现编码为 `{field: state}` 的结构化 JSON，每个 token 都可验证。相比自由文本，JSON 格式固定 → LLM 预测确定性高 → 梯度信号更干净。
3. **SPD（Structured Prediction Decomposition）**：3 组可学习 query tokens 通过 cross-attention 从 ViT 特征中分解提取不同方面信息，正交性约束减少冗余。

**第三幕：验证（So What）**

用表征学习领域标准评估（linear probe）证明 ViT 表征质量：

| 评估维度 | 结论 | 数据支撑 |
| --- | --- | --- |
| 表征质量 | SPD 在 LP 上 12-label F1/AUC 均为最优 | F1=0.9129（+2.84pp），AUC=0.8441（+9.18pp） |
| 跨域泛化 | CheXpert → NIH 排序完全一致 | NIH AUC=0.7121（+9.62pp） |
| 组件贡献 | 每一步递进都有正增益 | free-text → UMS: AUC +2.47pp; UMS → SPD: AUC +1.03pp |
| vs ViTP 风格 | Random mask 损害表征 | Random mask F1 低于 baseline（-2.42pp） |

### 7.4 Reviewer 可能的攻击点 & 预备回答

| 攻击点 | 回答策略 |
| --- | --- |
| "SPD 增益偏小（+1.03pp AUC）" | 12-label 口径下三步递进均正增益；4-group 版本增益已从 +0.72pp 提升到 +1.03pp；多 seed 实验验证一致性（待跑） |

### 8.4 新增对比实验设计（假设消融已验证）

消融递进链（A → A+UMS → A+UMS+SPD）已证明每个组件的独立贡献。以下对比实验从不同维度进一步强化论文论证。

#### Exp-C1: Projector 设计对比（Table 3 扩展行）

固定 frozen LLM + UMS-JSON 12-label，只替换 projector 模块，证明 SPD 优于其他 projector 设计。

| 对比组 | Projector | 证明什么 | 配置差异 |
| --- | --- | --- | --- |
| A+UMS（已有） | MLP + prefix tokens | baseline projector | spd.enabled=false |
| A+UMS+SPD（已有） | SPD cross-attention | SPD > MLP | spd.enabled=true |
| A+UMS+Q-Former | Q-Former（共享 cross-attn，无正交约束） | SPD 正交约束的价值 | 新增 Q-Former projector |
| A+UMS+HFP | HFP 多层融合 | SPD（横向分解）> HFP（纵向融合） | hfp.enabled=true, spd.enabled=false |

> Q-Former 对比的意义：SPD 和 Q-Former 都用 cross-attention query tokens，但 SPD 有多组独立 attention + 正交约束。如果 SPD > Q-Former，说明正交分解是关键而非 cross-attention 本身。
> 实现方式：将 SPD 的 num_groups 设为 1（单组 6 tokens，无正交约束）即可模拟 Q-Former 行为，无需新代码。

#### Exp-C2: LLM 规模对比（Scaling Analysis）

固定 ViT + SPD + UMS-JSON，替换不同规模的 frozen LLM，验证 LLM 语义空间质量对表征学习的影响。

| 对比组 | LLM | 参数量 | 证明什么 |
| --- | --- | --- | --- |
| SPD + Qwen2.5-0.5B | Qwen2.5-0.5B-Instruct | 0.5B | 小 LLM 是否足够 |
| SPD + Qwen2.5-1.5B（已有） | Qwen2.5-1.5B-Instruct | 1.5B | 当前配置 |
| SPD + Qwen2.5-3B | Qwen2.5-3B-Instruct | 3B | 更大 LLM 是否更好 |

> 预期：LLM 越大，语义空间越丰富，LP AUC 越高。但边际收益递减。
> 论文价值：(1) 证明 LLM 语义空间质量确实影响表征学习，(2) 1.5B 是性价比最优点（训练显存可控），(3) 回应 reviewer "为什么用小 LLM" 的质疑。
> 工作量：2 次预训练 + 2 次 LP。需确认 3B 模型在单卡上的显存是否可行（gradient_accumulation 可调大）。

#### Exp-C3: 监督信号粒度对比（Label Granularity）

固定 frozen LLM + UMS-JSON + SPD，改变监督标签数量，验证更多标签 → 更丰富梯度信号 → 更好表征。

| 对比组 | 标签数 | 证明什么 |
| --- | --- | --- |
| SPD 5-label | 5（CheXpert competition 标签） | 少标签下的表征质量 |
| SPD 7-label（已有，V10.1） | 7 | 当前主表配置 |
| SPD 12-label（消融链） | 12 | 更多标签是否更好 |

> 预期：标签越多，LLM 需要预测的 JSON 字段越多，ViT 需要编码的语义信息越丰富，LP 表征质量越好。
> 论文价值：证明 UMS-JSON 的可扩展性——增加标签不需要改架构，只需扩展 JSON schema。
> 工作量：1 次预训练（5-label）+ 1 次 LP。7-label 和 12-label 已有。

#### Exp-C4: 冻结 vs 训练 LLM（核心差异化实验）

固定 ViT + MLP projector + UMS-JSON 12-label，对比冻结 LLM 和训练 LLM。这是与 ViTP 差异化的核心实验。

| 对比组 | LLM 状态 | 证明什么 |
| --- | --- | --- |
| Frozen LLM（已有，A+UMS） | 冻结 | 稳定语义锚点 |
| Trainable LLM（新增） | 解冻，lr=1e-5 | 目标空间漂移的影响 |
| Trainable LLM + LoRA（新增） | LoRA 微调，rank=8 | 轻量训练 LLM 的折中 |

> 预期：Frozen > Trainable。训练 LLM 导致目标空间漂移，ViT 学到的表征不稳定。
> 论文价值：直接回答 "为什么冻结 LLM" 这个核心设计选择，也是与 ViTP 最关键的差异。
> 实现：需要在 vivid_model.py 中加 LLM unfreeze 选项。LoRA 需要 peft 库。
> 工作量：2 次预训练 + 2 次 LP。Trainable LLM 显存更大（需要存 LLM 梯度），可能需要减小 batch_size。
