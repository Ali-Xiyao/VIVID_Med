# 训练目标与监督设计

> 本文件合并了原 UMS/Verifier/FailureTaxonomy 设计理念与训练目标/SamplePolicy。
> 注意：V9 SVRL 实际只实现了 L_tok + L_cons（consistency loss）。L_rank/L_vdep/L_ans/verifier-guided policy 为设计蓝图，尚未实现。

## 1. UMS-JSON 设计理念

### 1.1 核心原则

- **凡是"可以从几何/元数据算出来的"，绝不让 LLM 编答案**
- **`null` 是"不可答"的正式答案**：字段缺失/不可判定必须显式 `null`，配套 `answerability=false`
- 输出被约束在 UMS-JSON schema 允许的 token 集合中

### 1.2 UMS 字段语义（v1.0）

| 字段 | 类型 | 语义 | null 含义 |
| --- | --- | --- | --- |
| modality | enum | CXR/CT/MRI | 不应为 null |
| anatomy | list[str] | 解剖区域 | 不应为 null |
| findings | dict | 每个 finding 含 state+score | state=null 表示不可判定 |
| study_view | enum | AP/PA/Lateral/... | 不可判定时为 null |
| laterality | enum | left/right/bilateral | 不可判定时为 null |
| answerability | dict | 字段级可答性 | — |

### 1.3 Verifier 设计理念（蓝图）

Verifier 不只是清洗器，而是进入训练算法的接口：

| verifier 结果 | 训练角色 | 进入的 loss |
| --- | --- | --- |
| pass & answerable | positive | L_tok + L_rank + L_vdep |
| pass & unanswerable | abstain | L_tok（目标为 null） |
| fail: unit/laterality | negative-only | L_rank |
| fail: schema_violation | drop | 不参与训练 |

> **实现状态**：V9 未实现 verifier-guided policy，所有样本统一进入 L_tok。

## 2. 训练目标

### 2.1 L_tok：Next-token prediction（主监督，已实现）

标准 cross-entropy，让模型生成 target JSON tokens。对 `answerable=false` 的字段，target 固定为 `null`。

### 2.2 L_cons：Augmentation Consistency（V9 新增，已实现）

对同一图像的两次不同 GFTM mask 的 forward pass，取 LLM hidden states 的 mean pooling，计算 cosine similarity loss：

`L_cons = 1 - cos(h1, h2)`，λ_cons = 0.1，warmup 2000 步后启用。

### 2.3 L_rank：Hard-negative ranking（蓝图，未实现）

对每条样本生成 hard negatives（格式合法但内容错），训练偏好约束。

Hard negative 类型：
- laterality flip：left↔right
- unit swap：mm↔cm
- measurement ×10/÷10
- confusable finding swap

### 2.4 L_vdep：Visual-dependence / anti-shortcut（蓝图，未实现）

- prompt corruption：语义不变，措辞变化 → 输出应一致
- image swap：prompt 不变，图像变化 → 输出应变化

### 2.5 L_ans：Answerability head（蓝图，未实现）

为每个字段预测可答性（二分类），推理时做 gating。

### 2.6 V9 实际总 Loss

```
L = L_tok + λ_cons × L_cons
```

其中 λ_cons = 0.1，前 2000 步 warmup 期间 λ_cons = 0。

## 3. Constrained JSON 生成

### 3.1 训练时

Constraint-aware Teacher Forcing（masked-CE）：
1. 根据 JSON grammar + 前缀计算合法 token 集
2. 将非法 token 的 logits 置为 -∞
3. 再做 softmax + cross entropy

### 3.2 推理时

使用 constrained decoding（同一套 grammar/FSM 与训练时一致）。

## 4. Failure Taxonomy（设计蓝图）

| failure_type | 含义 | 严重程度 |
| --- | --- | --- |
| schema_violation | JSON 不合法 | 高 |
| laterality_error | 左右判错 | 高（临床） |
| unit_error | 单位错误 | 高（临床） |
| measurement_hard_error | 数值偏差超阈值 | 高 |
| finding_confusion | 可混淆疾病判错 | 中 |
| missing_field | 缺失必填字段 | 中 |

> 这些 failure types 在 V9 中尚未作为训练信号使用，但可用于评测分析。
