# VIVID-Med 方案总览（V9 SVRL 当前版）

## 0. 一句话定位

**我们反过来用 LLM：不训练 LLM 去看医学图像，而是用冻结 LLM 作为"结构化语义解码器/监督空间"，训练 ViT 学到可迁移、可验证的医学视觉表征。**

## 0.1 论文故事线（Semantic Knowledge Distillation）

传统 one-hot 标签把疾病/属性当作彼此正交；冻结 LLM 的语义空间天然编码医学概念的相似性与共现拓扑。我们通过"结构化、受约束"的 token 对齐训练，让 **ViT 表征被蒸馏到 LLM 的语义流形上**，从而获得更强的跨域泛化与抗 shortcut 能力。

拆开理解：
1. **one-hot 的盲点**：标签把概念当作彼此独立的维度，只告诉模型"对/错"，不鼓励显式学到概念相似性与共现结构。
2. **LLM 的价值**：冻结 LLM 的表示空间里天然包含概念关系（距离/相似性/共现拓扑），可以视作一个"语义监督空间"。
3. **蒸馏的对象**：我们不是让 ViT 学会聊天，而是用结构化、受约束的 JSON token 目标去反向塑造 ViT 表征，使其经 projector 映射后能在该语义空间里"落到正确位置"。

> 主故事线：**LLM 是训练时的老师（监督空间），ViT 是最终交付物（产物）**。

## 0.2 最终交付物

- **产物 A（主产物：表征）**：一个更强的 **medical ViT backbone checkpoint**（+ 可复现训练配方）。推理时输出 embedding/feature map；按需求接分类/分割/回归/检索等下游 head（LLM 不参与推理）。
- **产物 B（可选加分：端到端结构化）**：在 ViT 上再训练一个不含 LLM 的轻量 `UMS head/decoder`，让推理时直接输出 UMS 字段。

## 1. 三个核心结论（全篇主线）

1. **LLM 是工具，ViT 是产品**：训练结束后丢掉 LLM，ViT-only 下游仍显著提升。
2. **结构化输出可验证，降低幻觉风险**：输出被约束在 UMS-JSON schema 内。
3. **模型真的看图（anti-shortcut 可量化）**：通过 image-swap / prompt-corruption / risk–coverage + 临床硬错误证明不是语言捷径。

### 1.1 三锤硬证据

- **ViT-only transfer**：丢掉 LLM 后，external test / transfer / low-data 仍显著提升
- **anti-shortcut**：image-swap / prompt-corruption
- **可靠性**：risk–coverage + clinical hard errors（laterality/unit/×10 等）

## 2. 三段式流水线

### (A) Offline：Verified Supervision 构建（数据轴）

公开数据（图像 + 标签/测量/元数据）→ **UMS-JSON**（含 answerability/uncertainty/provenance）→ **verifier** 输出 `pass/fail + failure_type + confidence + role`

### (B) Online：LLM-mediated Structured Alignment 训练（训练轴）

`image → ViT(train) → projector(train) → frozen LLM(forward，可导到输入) → JSON tokens`

V9 新增：GFTM（attention-guided token masking）+ FSA Consistency（冻结 LLM 语义锚点一致性）+ CMSA（CXR+CT 跨模态混合训练）

### (C) Eval：ViT-only 迁移 + 可靠性评测（证据轴）

- **ViT-only transfer**：丢掉 LLM，仅用 ViT encoder 接 head/下游框架
- **分类 transfer**：CheXpert → NIH external test
- **分割 transfer**：KiTS21（+ low-data 曲线 10%/25%/50%/100%）

## 3. 当前模型配置（V9 SVRL 实际使用）

### 3.1 Vision Encoder (ViT)

| 配置项 | 值 | 说明 |
|-------|-----|------|
| 模型 | `vit_base_patch16_224` | ViT-Base，patch 16，输入 224×224 |
| 预训练 | ImageNet-21k | timm 库加载 |
| 输出类型 | `all`（V9） | 所有 patch tokens + CLS |
| 参数量 | ~86M | **可训练** |

### 3.2 Projector

| 配置项 | 值 | 说明 |
|-------|-----|------|
| 结构 | 2-layer MLP | 768 → 1536 → 1536 |
| 前缀 tokens | 4（V9） | 可学习 prefix tokens |
| Dropout | 0.1 | |
| 参数量 | ~6M | **可训练** |

### 3.3 LLM (冻结)

| 配置项 | 值 | 说明 |
|-------|-----|------|
| 模型 | `Qwen/Qwen2.5-1.5B-Instruct` | 1.5B 参数 |
| 精度 | BFloat16 | |
| 状态 | **冻结，不训练** | requires_grad=False，但 forward 可导 |

### 3.4 V9 SVRL 特有组件

| 组件 | 说明 |
|------|------|
| GFTM | CLS-to-patch attention guided masking，mask_ratio 0.5→0.75 |
| FSA Consistency | 两次不同 mask 的 LLM hidden states cosine similarity，λ=0.1 |
| CMSA | CXR:CT = 7:3 混合训练，统一 UMS-JSON schema |
| ViTSegHead | 4 层 ConvTranspose2d，用于分割 transfer |

### 3.5 参数统计

| 组件 | 参数量 | 状态 |
|------|--------|------|
| ViT | ~86M | 可训练 |
| Projector | ~6M | 可训练 |
| LLM | ~1.5B | 冻结 |
| **总可训练** | **~92M** | |

## 4. 数据配置（V9）

| 数据集 | 用途 | 样本量 |
|--------|------|--------|
| CheXpert 30k | CXR 预训练 | 30,000 |

| NIH ChestX-ray14 | CXR external test | 25,596 |

## 5. 必做对照与"翻车预案"

- **必须对照：task-head baseline**（同数据同 backbone 直接接分类/分割/回归头训练）
- **Plan A（理想）**：在 external/low-data/可靠性上显著赢 task-head baseline
- **Plan B（可接受）**：task-head baseline 在同分布满数据更强，但你在 external/low-data/硬错误/拒答可靠性更强 → 主张写成"面向临床部署的鲁棒与安全表征"
