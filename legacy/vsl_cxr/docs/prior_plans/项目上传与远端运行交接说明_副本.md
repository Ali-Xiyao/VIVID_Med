# 项目上传与远端运行交接说明

更新时间：2026-07-03

本文用于把另一台机器上的深度学习项目上传到上海工程技术大学算力平台，并交给后续远端调试/运行。请把这份文档发给负责上传项目的人。

## 1. 重要安全规则

不要把真实登录密码写进 Markdown、脚本、README、聊天记录或邮件正文里。

本账号信息可以写：

```text
平台地址: 172.20.52.10
用户名: dqxy11
远端家目录: /ipfs/inspurfileset/home/dqxy/dqxy11
远端项目根目录: /ipfs/inspurfileset/home/dqxy/dqxy11/projects/xiyaowang
推荐远端项目命名: 见第 3 节
```

密码处理方式：

- 首次密码只由账号本人在终端里手动输入。
- 如果必须给别人临时登录，建议改成临时密码，用完立刻改掉。
- 更推荐使用 SSH key。对方只把公钥 `.pub` 发过来，不发私钥，不发密码。
- 如果某个密码已经出现在聊天或文档里，建议尽快在平台 Web 页面修改密码。

## 2. 远端目录规范

所有个人项目统一放在：

```bash
/ipfs/inspurfileset/home/dqxy/dqxy11/projects/xiyaowang
```

登录后可以用快捷命令进入：

```bash
xw
```

推荐每个项目按下面结构放置。假设项目名叫 `my_project`：

```text
~/projects/xiyaowang/
  code/my_project/             # 代码仓库，只放源码和轻量配置
  data/my_project/raw/          # 原始数据
  data/my_project/processed/    # 预处理后的数据
  outputs/my_project/           # 预测、评估、可视化等输出
  checkpoints/my_project/       # 模型权重和 checkpoint
  logs/my_project/              # slurm 输出、训练日志、tensorboard 日志
  env/my_project/               # requirements、environment.yml、安装记录
  docs/my_project/              # 运行说明、实验记录、数据说明
```

不要把大数据、模型权重、日志和输出混在代码仓库里。代码放 `code/`，数据放 `data/`，结果放 `outputs/` 或 `checkpoints/`。

如果代码里默认读取项目内的 `data/`、`outputs/`、`checkpoints/`，上传后可以在代码目录里创建软链接：

```bash
cd ~/projects/xiyaowang/code/my_project
ln -sfn ~/projects/xiyaowang/data/my_project data
ln -sfn ~/projects/xiyaowang/outputs/my_project outputs
ln -sfn ~/projects/xiyaowang/checkpoints/my_project checkpoints
ln -sfn ~/projects/xiyaowang/logs/my_project logs
```

## 3. 项目命名和文件要求

项目名建议只使用英文、数字、下划线或短横线：

```text
good: hioapd, rarecardio_v1, sleep-apnea-dl, mm_signal_202607
bad: 新建文件夹, final最终版, test 1, a/b, 含空格的名字
```

上传前项目里至少应包含：

```text
README.md 或 RUN.md               # 怎么安装、怎么跑
requirements.txt 或 environment.yml # Python 依赖
scripts/ 或 run.sh                 # 推荐入口脚本
configs/                           # 训练配置，如果项目使用配置文件
src/ 或 项目源码目录                # 主代码
```

上传前请清理这些不该传的内容：

```text
.git/                    # 如果只是交付代码包，可不传；若要保留版本历史另说
__pycache__/
.pytest_cache/
.mypy_cache/
.ruff_cache/
.DS_Store
.vscode/
.idea/
.venv/
venv/
env/
node_modules/
wandb/                   # 历史日志太大时不要传
runs/                    # 历史 tensorboard 日志太大时不要传
outputs/                 # 旧输出，除非明确需要复现实验
checkpoints/             # 大权重，除非明确需要继续训练或评估
*.pt *.pth *.ckpt *.safetensors  # 大模型文件建议单独传到 checkpoints/
```

如果项目需要大权重或大数据，请单独说明每个文件的用途，不要直接塞在代码包里。

## 4. 推荐上传方式：SSH key + rsync

### 4.1 在项目所在机器生成 SSH key

macOS/Linux：

```bash
ssh-keygen -t ed25519 -C "sues-hpc-dqxy11-project-upload" -f ~/.ssh/sues_hpc_dqxy11
```

Windows PowerShell：

```powershell
ssh-keygen -t ed25519 -C "sues-hpc-dqxy11-project-upload" -f $env:USERPROFILE\.ssh\sues_hpc_dqxy11
```

这会生成两份文件：

```text
sues_hpc_dqxy11      # 私钥，必须留在自己机器，不要发给任何人
sues_hpc_dqxy11.pub  # 公钥，可以发给账号本人添加到服务器
```

### 4.2 把公钥加入服务器

方式 A：让账号本人添加公钥。对方把 `sues_hpc_dqxy11.pub` 的内容发给账号本人，然后账号本人在服务器上执行：

```bash
mkdir -p ~/.ssh
chmod 700 ~/.ssh
cat >> ~/.ssh/authorized_keys
# 粘贴对方发来的 .pub 公钥，回车，然后 Ctrl-d 结束
chmod 600 ~/.ssh/authorized_keys
```

方式 B：如果对方已经知道临时密码，可以自己执行：

```bash
ssh-copy-id -i ~/.ssh/sues_hpc_dqxy11.pub dqxy11@172.20.52.10
```

Windows PowerShell 没有 `ssh-copy-id` 时，可以先打印公钥：

```powershell
type $env:USERPROFILE\.ssh\sues_hpc_dqxy11.pub
```

然后把输出发给账号本人按方式 A 添加。

### 4.3 配置 SSH 简写

macOS/Linux 在 `~/.ssh/config` 追加：

```sshconfig
Host sues-hpc
    HostName 172.20.52.10
    User dqxy11
    IdentityFile ~/.ssh/sues_hpc_dqxy11
    IdentitiesOnly yes
```

Windows PowerShell 对应文件一般是：

```text
C:\Users\<你的用户名>\.ssh\config
```

测试：

```bash
ssh sues-hpc
```

能直接登录就说明免密配置成功。

## 5. 上传代码

假设本地项目目录是：

```text
/path/to/my_project
```

先在远端建目录：

```bash
ssh sues-hpc 'mkdir -p ~/projects/xiyaowang/{code,data,outputs,checkpoints,logs,env,docs}'
ssh sues-hpc 'mkdir -p ~/projects/xiyaowang/code/my_project'
```

推荐用 `rsync` 上传代码：

```bash
rsync -avzP \
  --exclude '.git/' \
  --exclude '__pycache__/' \
  --exclude '.pytest_cache/' \
  --exclude '.mypy_cache/' \
  --exclude '.ruff_cache/' \
  --exclude '.DS_Store' \
  --exclude '.vscode/' \
  --exclude '.idea/' \
  --exclude '.venv/' \
  --exclude 'venv/' \
  --exclude 'env/' \
  --exclude 'node_modules/' \
  --exclude 'wandb/' \
  --exclude 'runs/' \
  --exclude 'outputs/' \
  --exclude 'checkpoints/' \
  /path/to/my_project/ \
  sues-hpc:~/projects/xiyaowang/code/my_project/
```

如果不想配置 SSH 简写，也可以直接写完整地址：

```bash
rsync -avzP /path/to/my_project/ dqxy11@172.20.52.10:~/projects/xiyaowang/code/my_project/
```

第一次上传前可以 dry run，确认不会传错目录：

```bash
rsync -avznP /path/to/my_project/ sues-hpc:~/projects/xiyaowang/code/my_project/
```

## 6. 上传数据、权重和输出

数据建议放这里：





## 9. 环境说明

远端已经有基础 Conda：

```bash
source ~/miniforge3/etc/profile.d/conda.sh
conda activate dl310
python --version
```

当前通用环境：

```text
环境名: dl310
Python: 3.10.20
已装: numpy, pandas, scipy, scikit-learn, matplotlib, seaborn, opencv, h5py,
      pyarrow, openpyxl, tensorboard, transformers, datasets, accelerate,
      safetensors, timm, wandb, torch, torchvision
```

注意：`dl310` 里的 PyTorch 当前按之前记录是 CPU 版。GPU 节点已经实测为 A800 80GB、Driver 590.48.01、CUDA 13.1；如果项目要用 GPU 训练，上传后需要先在 GPU bash 里确认并安装合适的 CUDA PyTorch 或创建项目专用环境。

不要在登录节点上跑长训练。登录节点只用于上传、编辑、查看队列、准备脚本。训练要进 CPU/GPU 计算资源里跑。

## 12. 一个最小 Slurm GPU 脚本模板

保存为 `~/projects/xiyaowang/code/my_project/scripts/train_1gpu.slurm`：

```bash
#!/bin/bash
#SBATCH --job-name=my_project_train
#SBATCH --partition=gpu
#SBATCH --nodes=1
#SBATCH --ntasks=1
#SBATCH --cpus-per-task=4
#SBATCH --gres=gpu:1
#SBATCH --mem=64G
#SBATCH --account=dqxy
#SBATCH --output=/ipfs/inspurfileset/home/dqxy/dqxy11/projects/xiyaowang/logs/my_project/%j.out
#SBATCH --error=/ipfs/inspurfileset/home/dqxy/dqxy11/projects/xiyaowang/logs/my_project/%j.err

set -euo pipefail

source ~/miniforge3/etc/profile.d/conda.sh
conda activate dl310

cd ~/projects/xiyaowang/code/my_project
mkdir -p ~/projects/xiyaowang/logs/my_project
mkdir -p ~/projects/xiyaowang/outputs/my_project
mkdir -p ~/projects/xiyaowang/checkpoints/my_project

echo "job id: ${SLURM_JOB_ID}"
echo "host: $(hostname)"
echo "CUDA_VISIBLE_DEVICES=${CUDA_VISIBLE_DEVICES:-unset}"
nvidia-smi

python train.py \
  --data_dir ~/projects/xiyaowang/data/my_project \
  --output_dir ~/projects/xiyaowang/outputs/my_project \
  --checkpoint_dir ~/projects/xiyaowang/checkpoints/my_project
```

提交：

```bash
sbatch ~/projects/xiyaowang/code/my_project/scripts/train_1gpu.slurm
```

查看：

```bash
squeue -u dqxy11
tail -f ~/projects/xiyaowang/logs/my_project/<JOBID>.out
```

## 13. 如果只想打包后上传

在项目所在机器打包：

```bash
cd /path/to
tar \
  --exclude='my_project/.git' \
  --exclude='my_project/__pycache__' \
  --exclude='my_project/.DS_Store' \
  --exclude='my_project/.venv' \
  --exclude='my_project/outputs' \
  --exclude='my_project/checkpoints' \
  -czf my_project_20260703.tar.gz my_project
```

上传：

```bash
scp my_project_20260703.tar.gz sues-hpc:~/projects/xiyaowang/code/
```

远端解压：

```bash
ssh sues-hpc
cd ~/projects/xiyaowang/code
tar -xzf my_project_20260703.tar.gz
```

大项目更推荐 `rsync`，因为 `tar.gz` 中断后通常要重新传。
