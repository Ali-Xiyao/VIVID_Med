# RCSD-CXR experiment plan

**Final status:** terminal NO-GO on 2026-07-23. G2 rejected the fusion
contribution and G3 rejected the simplified field-anchor contribution. This
plan is retained as prospective design history and no longer authorizes runs.
See `../docs/RCSD_CXR_terminal_gate_result_20260723.md`.

**Problem:** Distill reliable structured report semantics into a deployable
chest-radiograph encoder without treating missing or conflicting parser output
as image truth.

**Method thesis:** A calibrated multi-source report posterior and an
equal-budget field-anchored 4x2 query interface should improve a ViT encoder
over the historical hard-UMS 4x2-SPD baseline.

**Date:** 2026-07-23

## Claim map

| Claim | Minimum convincing evidence | Linked blocks |
| --- | --- | --- |
| C1: calibrated structured supervision is better than a single hard source | Independent report-gold F1/NLL/ECE improvement, followed by a controlled posterior-only visual gain | B1, B2 |
| C2: field anchoring adds value without increasing the query-token budget | Anchor-only and full RCSD beat equal-budget 4x2 SPD under one data/compute/checkpoint protocol | B2, B3 |

Anti-claims to rule out: gains are only label smoothing, only extra parameters,
only teacher scale, only one finding/seed, or selected using an external test.

## Paper storyline

- Main paper must prove label-posterior validity, controlled MIMIC-only method
  gain, low-data benefit, calibration, and external non-inferiority.
- Appendix may contain Qwen3.5 teacher-size sensitivity, multi-institution
  scaling, second backbone, and semantic geometry.
- CT/pathology, causal localization, and post-hoc test-driven rescue are cut.

## Experiment blocks

### B0: Identity and runnable-surface gate

- Claim tested: none; this protects every later claim.
- Dataset/split: MIMIC official train/validate only; test rejected.
- Required evidence: local/remote identity, canonical frontal manifest,
  patient disjointness, source lineage, license/DUA ledger, test seal, code and
  environment hashes.
- Success: every G0 item in the active protocol passes machine-readably.
- Failure: stop all training.
- Priority: MUST-RUN.

### B1: Supervision validity

- Claim tested: C1.
- Compared systems: strongest single reproducible source, static average,
  calibrated RCSD posterior.
- Metrics: macro-F1, state F1, NLL, ECE, reliability-correctness AUROC.
- Success: active-protocol G2 thresholds on an independent report-gold set.
- Failure: drop fusion and use the best single source; do not add parsers.
- Target: main Table 2/Figure 2.
- Priority: MUST-RUN.

### B2: Controlled mechanism and historical baseline

- Claim tested: C1 and C2.
- Systems: A+UMS, A+UMS+SPD 4x2, posterior+SPD, hard+field-anchor,
  full RCSD.
- Run order: 256-row overfit, 5k learnability, 20k pilot, full MIMIC seed 0.
- Primary selection metric: validation structured NLL from one checkpoint.
- Success: G1 through G4 in order; select the simplest surviving variant.
- Failure: stop at the first failed gate; implementation-only repair is
  allowed at learnability gates.
- Target: Tables 3 and 6.
- Priority: MUST-RUN.

### B3: Paper-level validation

- Claim tested: cross-institution and low-data transfer of the simplest
  surviving method.
- Tasks: CheXpert probes/fine-tuning, 1/5/10/25/100 percent low-data,
  NIH frozen transfer, previously-inspected VinDr replication, calibration and
  descriptive subgroups.
- Statistics: three seeds, patient-level paired/hierarchical bootstrap,
  FDR-controlled per-finding analysis.
- Success: G5 criteria; no external domain breaches the non-inferiority bound.
- Target: Tables 3-5 and 8, Figures 3-4.
- Priority: MUST-RUN after B2 survives.

### B4: Qwen3.5 teacher-size sensitivity and scale

- Claim tested: the method does not rely on a single teacher scale.
- Available frozen teachers: Qwen3.5-0.8B, 2B, 4B, and 9B.
- Primary teacher: Qwen3.5-2B, selected before visual results because it is the
  closest bounded replacement for the historical small teacher.
- Sensitivity teachers: 0.8B, 4B, 9B, run sequentially with identical
  templates and prototype contracts only after G5.
- Success: qualitative direction is not unique to the largest teacher; size is
  not selected using test metrics.
- Failure: report teacher dependence; do not rescue the main method by teacher
  scaling.
- Target: supplement teacher-sensitivity table.
- Priority: NICE-TO-HAVE / user-requested conditional queue item.

## Run order and milestones

| Milestone | Runs | Decision gate | Cost | Risk |
| --- | --- | --- | --- | --- |
| M0 | code/env/model/data audit; canonical manifest | G0 | CPU, hours | incomplete identity or adapters |
| M1 | historical config audit; 256-row overfit | G1 learnability | <1 GPU-day | old code semantics mismatch |
| M2 | independent gold label-model evaluation | G2 | CPU, hours | gold resource absent |
| M3 | 5k then 20k controlled variants | G3 | several GPU-days | supervision/anchor no gain |
| M4 | full MIMIC seed 0 | G4 | multi-day GPU | external degradation |
| M5 | final three seeds and downstream matrix | G5 | multi-week serial GPU | one-A800 throughput |
| M6 | Qwen3.5 0.8B/4B/9B sensitivity; optional Track B | G6 | conditional | storage/time and teacher confound |

## Compute and data budget

- Final execution hardware: retained allocation `3066`
  (`bash-gpu01-64g4c`), one A800 80GB, four CPUs, 64GB.
- Sharing: unrelated GPU work is preserved; RCSD GPU stages wait for an
  explicit headroom guard and run one at a time.
- Storage: about 185.5 GiB quota headroom at the last verified snapshot.
- Model weights are referenced in place and never copied into project outputs.
- Checkpoints are retention-limited: best plus last during development; only
  frozen finalists retained after a gate.
- Biggest bottleneck: missing end-to-end training/label/evaluation adapters,
  followed by one-GPU serial throughput.

## Risks and mitigations

- Missing report-gold resource: do not substitute image test labels; block G2.
- Shared GPU spikes: wait instead of killing another process; record live GPU
  facts at launch and completion.
- Teacher ambiguity: lock Qwen3.5-2B primary before results; other sizes are
  conditional sensitivity only.
- Legacy leakage: migrate only audited representation-learning components and
  tests; no BiVES/MORPH code path.
- Storage pressure: no duplicate weights/data; checkpoint retention and quota
  guard before every full run.

## Terminal decision

- G2 fusion: NO-GO because NLL degraded 6.98% relative to CheXbert.
- G3 field anchoring: NO-GO because NLL improved only 0.046% and macro-F1
  improved only 0.0648 pp.
- G4-G6 and every Qwen3.5 size sensitivity run are cancelled.
- External tests, including CheXlocalize test, remain unopened by this route.
